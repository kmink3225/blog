# 질문 응답 가이드

사용자가 특정 주제에 대해 질문하면, `guides/info-search.md`의 병렬 탐색 프로토콜을 **먼저 실행**한다. 블로그 포스트, 교재 소스, agent 사전지식을 **모두 동시에** 탐색한 뒤, 3개 레이어의 결과를 통합하여 답변한다.

## 탐색 절차

`guides/info-search.md`의 병렬 탐색 프로토콜을 그대로 따른다. Q&A라고 해서 교재 탐색을 생략하지 않는다.

## 표준 답변 템플릿 (Mandatory Output Format)

모든 `/qa` 답변은 탐색의 투명성과 정확성을 위해 반드시 아래 형식을 서두에 포함해야 한다.

### 1. 의도 분석 및 키워드 확장
- **의도**: 사용자의 질문 이면에 숨겨진 기술적 맥락과 근본적인 목적을 분석하여 기술한다.
- **확장 키워드**: 에이전트 지식을 활용해 도출한 연관 용어, 기술 스택, 관련 개념 등을 나열한다.

### 2. 3-Layer 병렬 탐색 결과
- **Layer 1 (블로그 포스트)**: `docs/blog/posts/` 탐색 결과 및 관련 파일 경로를 나열한다.
- **Layer 2 (교재 소스)**: `docs/book/` 내의 관련 챕터 및 인용 가능 정보를 나열한다.
- **Layer 3 (에이전트 사전지식)**: 에이전트가 보유한 관련 기술 지식의 요점을 나열한다.

## 답변 구성

상기 탐색 결과를 바탕으로 구체적인 답변을 작성한다. 각 레이어에서 가져온 정보를 명확히 구분한다:
...
| 소스 | 답변에서의 표시 | 충돌 시 신뢰도 |
|------|----------------|---------------|
| 블로그 포스트 | 파일 경로 제시 | 최우선 |
| 교재 소스 (`docs/book/`) | `(저자, 연도, Ch.N)` 인용 | 2순위 |
| agent 사전지식 | "블로그에 아직 없다" 명시 | 3순위 |

**충돌 시 신뢰도**는 "탐색을 멈추는 순서"가 아니다. 3개 레이어를 모두 탐색한 결과, 내용이 서로 다를 때 어느 쪽을 채택하는지의 기준이다.

## 콘텐츠 gap 발견 시

| 상황 | 행동 |
|---|---|
| 3개 레이어 모두에서 관련 정보 발견 | 통합하여 답변. 각 소스를 구분 표시 |
| 블로그에 있고 교재에도 있음 | 블로그 기반 답변 + 교재로 깊이 보강 |
| 블로그에 없지만 교재에 있음 | 교재 기반 답변 + "블로그에 아직 없다. 작성할까?" 제안 |
| 블로그·교재 모두 없음 | 사전지식 답변 + "블로그·교재 모두 없다" 명시 + 작성 제안 |
| `index.qmd`에 placeholder(1111-11-11)로만 존재 | "이 주제는 계획되어 있지만 아직 작성되지 않았다" + 탐색된 정보로 답변 + 작성 제안 |

## 주의사항

- **블로그 내용과 일반 지식을 혼동하지 않는다.** 블로그에 작성된 내용을 인용할 때는 파일 경로를 명시한다.
- **블로그의 용어와 표기법을 따른다.** 블로그에서 "한다 체"를 사용하더라도 답변은 대화체로 한다. 다만 블로그 내용을 인용할 때는 원문 그대로 제시한다.
