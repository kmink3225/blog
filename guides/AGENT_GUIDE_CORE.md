# AGENT_GUIDE_CORE — 항상-온 규칙 (Always-On Rules)

## 실행 체크포인트 (Claude Code 행동 교정)

> Claude Code는 가이드를 형식적으로 읽고 단계를 압축·생략하는 경향이 있다.
> 아래 체크포인트는 이를 방지하기 위한 **출력 의무**이다.

### CP-1: 가이드 로드 확인

skill 가이드(write-post.md 등)를 Read 도구로 읽은 후, 로드한 파일 목록을 한 줄로 출력한다.

```
예: "가이드 로드: CORE + write-post.md + Statistics/GUIDE.md + info-search.md"
```

"이미 알고 있으므로 생략"은 허용하지 않는다. Read 도구 호출 기록이 현재 세션에 없으면 반드시 읽는다.

### CP-2: Step 명시

skill 가이드의 각 Step을 실행할 때, 해당 Step 번호와 요약을 사용자에게 보이는 텍스트로 출력한다.

```
예: "Step 3 (중복 판단): 'Leibniz' Grep → 기존 포스트 없음, 신규 작성 진행"
```

여러 Step을 한 문장으로 뭉개거나, 암묵적으로 건너뛰지 않는다.

### CP-3: Self-Check 출력

작업 완료 보고 **직전에** 아래 Self-Check 항목을 체크리스트 형태로 **반드시 출력**한다.
내부적으로만 점검하고 출력을 생략하면 안 된다. 위반 항목이 발견되면 보고 전에 수정한다.

## 절대 규칙 5개

1. **한다 체** — `~한다/~이다/~된다` (O: `MLE는 로그가능도를 최대화하여 구한다` / X: `~로 구합니다`)
2. **수동 번호 금지** — `number-sections: true` 자동 부여 (O: `## 개요` / X: `## 1. 개요`)
3. **Category GUIDE 필수 로드** — 카테고리 포스트 작성·수정 전 반드시 해당 `GUIDE.md` 읽기
4. **index.qmd 업데이트** — 새 포스트 작성 후 반드시 해당 카테고리 `index.qmd`에 링크 추가
5. **이모지 사용 금지** — `.qmd` 콘텐츠에 이모지 넣지 않음

## YAML 헤더

```yaml
---
title: "제목"
subtitle: "부제목"
description: |
  1-3문장 설명.
categories:
  - Category1
author: Kwangmin Kim
date: MM/DD/YYYY   # 예: 03/28/2026. 미작성 글은 1111-11-11
---
```

- `format`, `toc`, `code-fold` 등은 `_metadata.yml` 상속, 생략
- `date` 형식: `MM/DD/YYYY` (index.qmd의 `YYYY-MM-DD`와 다름)

## 콘텐츠 스타일

| 항목 | 규칙 |
|------|------|
| 한다 체 | `~한다`, `~이다`, `~된다` |
| 경어체 | `~합니다`, `~입니다`, `~하세요` 금지 |
| 이모지 | `.qmd` 내 사용 금지 |
| 수동 번호 | 헤더에 `## 1. 개요` 형식 금지 |
| 분량 | 500줄 내외 (400~600줄). 200줄 미만은 깊이 부족, 800줄 이상은 독자 이탈 |
| 섹션 수 | 4~8개 (`##` 수준). 코드 블록 포함 시 코드+설명 합쳐 500줄 |
| 수식 공백 | 인라인: `$` 양쪽 공백 필수 (O: `모수 $\theta$ 를` / X: `모수$\theta$를`) |
| 블록 수식 | `$$수식$$` |
| 교재 인용 | `(저자, 연도, Ch.N)` |
| 줄 개행 | 문단 내 줄바꿈 필요 시 줄 끝 공백 2칸. 빈 줄 문단 구분·코드블록·YAML·callout·bullet 내부는 불필요 |

## 콘텐츠 품질 원칙

모든 서술은 **주장 → 근거 → 해석** 구조를 따른다.

| 점검 항목 | 질문 | 미충족 시 조치 |
|-----------|------|---------------|
| 주장 → 왜 | "왜 그게 문제인가?"에 답하는가 | 시나리오 또는 반사실적 비교 추가 |
| 숫자 → 해석 | 숫자 뒤에 "so what"이 있는가 | 실무적 임팩트로 번역 |
| 사례 → 연결 | 이 사례가 왜 여기 있는지 독자가 아는가 | 일반론을 증명하는 구조로 재배치 |
| 나열 → 흐름 | 목록이 단순 나열인가, 논리 흐름인가 | 항목 간 인과·순서 관계 명시 |
| 추상 → 직관 | 수식·가정·알고리즘 뒤에 비유가 있는가 | 일상어 비유 또는 반사실 시나리오 추가 |

자주 발생하는 오류 패턴(경어체 혼입, 수동 번호, 수식 공백 누락, 콘텐츠 축소)의 Bad→Corrected 예시는 `guides/common-mistakes.md` 참조.

## index.qmd 링크 패턴

index.qmd마다 형식이 다르다. **반드시 기존 항목 패턴을 읽고 동일하게** 작성한다.

| 패턴 | 카테고리 | 형식 |
|------|----------|------|
| A: 번호 목록 | Agent | `1.  [텍스트](./서브폴더/파일.qmd)` |
| B: 날짜 bullet | Engineering, Statistics, Governance | `* YYYY-MM-DD, [텍스트](./파일.qmd)` |
| C: 순수 목록 | Math, Deep_Learning | 링크 있는 항목과 없는 항목 혼재 |
| D: 구조화 학습 경로 | Experimentation, Machine_Learning | 설명 + 중첩 bullet |

- 상대경로: 같은 폴더 `./파일.qmd` / 서브폴더 `./서브폴더/파일.qmd` / 다른 카테고리 `../Other/파일.qmd`
- index.qmd 날짜는 `YYYY-MM-DD` (YAML 헤더의 `MM/DD/YYYY`와 다름에 주의)
- 파일이 아직 없어도 논리적으로 필요한 주제는 **placeholder 링크**로 포함한다. 파일 생성 시 자동 연결된다.
- 하나의 `.qmd`가 여러 카테고리에 해당하면: 물리적 위치는 주 카테고리에 두고, 관련 카테고리 index.qmd에도 `../OtherCategory/파일.qmd`로 **크로스 링크**를 건다. YAML `categories`에도 모두 나열한다.
- 여러 분야가 융합되는 주제(**Multi-disciplinary**)는 주 카테고리 GUIDE를 따르고, 관련 카테고리 GUIDE에서 용어·표기법·교재를 추가 확인한다.

## Stop-and-Ask 트리거

| 트리거 | 임계값 |
|--------|--------|
| 파일 생성 | **3개 이상** → 목록 보여주고 승인 후 진행 |
| 파일 삭제·이동 | **1개라도** → 대상·사유 보여주고 승인 후 진행 |
| index.qmd 패턴 판별 불가 | 즉시 사용자에게 질문 |
| 카테고리 분류 모호 | 후보 나열 후 선택 요청 |
| 가이드 파일 부재 | 생략 후 사용자에게 알림 |

## Fallback

| 상황 | 대응 |
|------|------|
| GUIDE.md 없음 | 로드 생략 + 사용자 알림 |
| index.qmd 패턴 불명 | 기존 항목 2~3개 인용 후 확인 요청 |
| 교재 파일 없음 | 교재 없이 진행 + 미참조 명시 |
| 판단 불가 | 작업 중단 + 상황 설명 + 지시 요청 |

## Post-Task 기본 동작 (Default: /publish 자동 실행)

**모든 태스크 완료 후 `/publish`를 자동으로 실행한다.** 별도 지시가 없어도 아래 절차를 이어서 수행한다.

```
태스크 완료 (write / fix / convert / series / audit / reindex 등)
    ↓
Self-Check 통과
    ↓
작업 결과 보고
    ↓
.\scripts\render-changed.ps1  ← 변경된 .qmd HTML 렌더링
    ↓
guides/publish.md 로드 → git add → commit → push → 충돌 해결
```

**예외 — /publish를 실행하지 않는 경우**:

- `/qa` (파일 변경 없음)
- `/log` (파일 변경 없음)
- 사용자가 명시적으로 "push 하지 마" / "commit 하지 마"라고 지시한 경우

## Self-Check (완료 직전 반드시 점검)

- [ ] 절대 규칙 5개 준수 (한다 체, 수동 번호 금지, Category GUIDE 로드, index 업데이트, 이모지 금지)
- [ ] YAML 필수 필드 (`title`, `description`, `categories`, `author`, `date`) 모두 있음
- [ ] `date` 형식 `MM/DD/YYYY`
- [ ] index.qmd 링크 패턴이 기존과 동일
- [ ] 수식 `$...$` 양쪽 공백 있음
- [ ] 포스트 분량 400~600줄 범위
- [ ] 모든 주장·숫자·수식 뒤에 "왜/so what/직관" 해석이 있음

## Boundaries

**할 수 있는 것**

- 블로그 포스트 작성, 수정, 보강
- index.qmd에 링크 추가 (placeholder 포함)
- 교재 소스(`docs/book/`) 참조 및 블로그 스타일로 재작성
- 질문에 대해 블로그 → 교재 → 사전지식 순서로 답변
- 크로스 카테고리 링크 추가

**하면 안 되는 것**

- 기존 포스트 내용 삭제·축소
- 교재 원문 그대로 복붙 (반드시 한다 체로 재작성)
- 블로그에 없는 내용을 블로그에서 읽은 것처럼 인용
- Category GUIDE.md를 읽지 않고 해당 카테고리 포스트 작성
- Summary MD만 읽고 포스트 작성 (Full MD 확인 필수)
- 읽지 않은 교재 구간을 원문에서 확인한 것처럼 서술
- 헤더에 수동 번호 붙이기

**권한 제한**

- `_quarto.yml`, `_metadata.yml`, `styles.css` 등 프로젝트 설정 파일 수정 금지
- 사용자 확인 없이 파일 3개 이상 생성 금지 (→ Stop-and-Ask)
