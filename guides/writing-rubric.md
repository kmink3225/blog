# 논리적 글쓰기 채점 루브릭

---

## 채점 체계 개요

| 단계 | 기준 | 방식 |
|------|------|------|
| **Gate** | R1 Stance | Pass / Soft Fail / Hard Fail — Hard Fail 시 채점 중단 |
| **Score** | R2~R7 | 각 5점 만점 × 6개 = **30점 만점** |

---

## R1 — Stance (입장 명확성) [Gate]

주장이 글 전체를 지탱하는 기둥이다. 이게 흔들리면 나머지 채점이 무의미하다.

| 판정 | 기준 | 처리 |
|------|------|------|
| **Pass** | 주장이 명확하고 한 문장으로 정리됨 | R2~R7 채점 진행 |
| **Soft Fail** | 주장은 있으나 모호하거나 양면적 | R1 피드백만 제공 → 재제출 → R2~R7 채점 진행 |
| **Hard Fail** | 주장 자체가 없거나 주제와 무관 | 채점 중단 → 전면 재작성 요구 |

**레벨별 Pass 기준:**

| 레벨 | Pass 기준 |
|------|----------|
| Lv.1~2 | 한쪽 입장이 보이면 Pass |
| Lv.3~4 | Thesis가 첫 문단에 명시되어야 Pass |
| Lv.5~6 | Thesis + 논거 방향 예고까지 있어야 Pass |
| Lv.7 | Thesis + 학술적 기여(contribution) 명시까지 |

---

## R2 — Focus (주제 일관성)

| 점수 | 기준 |
|------|------|
| 5 | 모든 문장이 thesis에 직접 기여, 불필요한 문장 없음 |
| 4 | 1~2문장이 주변부적이나 전체 흐름에 영향 없음 |
| 3 | 한 문단이 thesis와 느슨하게 연결됨 |
| 2 | 주제에서 벗어난 문단이 존재, 독자가 혼란 |
| 1 | 글의 절반 이상이 thesis와 무관 |

---

## R3 — Syntax (문법/언어 정확성)

**모드에 따라 평가 기준이 달라진다.**

| 모드 | 적용 조건 | R3 평가 대상 |
|------|----------|------------|
| **일반 모드** | 퇴고 가능한 글 (리포트, 논문, 보고서) | 오타·띄어쓰기·문법·어휘 모두 |
| **Timed 모드** | 시간 제한 조건 (TOEFL, GRE, 논술 시험, 세션 내 즉석 작성) | 어휘 범위·문법 구조 다양성만. 오타·띄어쓰기 감점 없음 |

**일반 모드 기준:**

| 점수 | 기준 |
|------|------|
| 5 | 오류 없음, 어휘 선택이 정확하고 자연스러움 |
| 4 | 사소한 오류 1~2개, 의미 전달에 영향 없음 |
| 3 | 오류가 산발적으로 있으나 의미는 파악 가능 |
| 2 | 오류가 빈번하여 독자가 의미를 재해석해야 함 |
| 1 | 오류가 글 전반에 퍼져 의미 전달 자체가 어려움 |

**Timed 모드 기준:**

| 점수 | 기준 |
|------|------|
| 5 | 다양한 문법 구조 구사, 고급 어휘 정확하게 사용 |
| 4 | 문법 구조 다양, 어휘 선택 대체로 적절 |
| 3 | 단순 문장 구조 반복, 어휘 범위가 좁음 |
| 2 | 문법 오류가 의미 전달을 방해 |
| 1 | 기초 문법 오류로 의미 파악 어려움 |

*레벨별 적용: Lv.1~2는 기초 문법, Lv.5~7은 어휘 정확성·어조(tone) 포함*

---

## R4 — Cohesion (유기성/연결성)

| 점수 | 기준 |
|------|------|
| 5 | 문장↔문장, 문단↔문단이 자연스럽게 이어짐. 연결어가 과하지 않고 논리 자체로 흐름 형성 |
| 4 | 연결이 대체로 자연스러우나 1곳에서 약간의 비약 |
| 3 | 연결어는 있으나 논리적 연결이 피상적, 문단 간 점프가 느껴짐 |
| 2 | 문단 간 연결이 끊기거나 순서가 부자연스러움 |
| 1 | 문장들이 나열 수준, 논리적 흐름 없음 |

---

## R5 — Structure (논리 전개 구조)

| 점수 | 기준 |
|------|------|
| 5 | Claim→Theory→Example→Action/Recommendation→Wrap-up 완성, 각 단계가 명확 |
| 4 | 구조가 있으나 한 단계가 약하거나 간략 |
| 3 | 일부 단계 누락, 전개가 불균형 |
| 2 | 구조의 흔적은 있으나 전개 방식이 불분명 |
| 1 | 구조 없음, 생각의 흐름대로 나열 |

*레벨별 적용: Lv.1~2는 Claim+Example만 요구, Lv.5 이상은 전체 5단계 요구*

---

## R6 — Depth (논거 독립성/심화)

| 점수 | 기준 |
|------|------|
| 5 | Supporting idea 2개 이상, 독립적이거나 심화 종속적. 각 논거가 thesis를 다른 각도에서 강화 |
| 4 | 논거 2개 이상이나 한 논거가 다른 논거의 반복 수준 |
| 3 | 논거가 1개이거나 2개지만 사실상 동일한 주장 |
| 2 | 논거가 있으나 주장을 뒷받침하지 못하고 표면적 |
| 1 | 논거 없음, 주장만 반복 |

---

## R7 — Flow (가독성/독자 경험)

| 점수 | 기준 |
|------|------|
| 5 | 처음부터 끝까지 멈춤 없이 읽힘. 독자가 "다음이 뭐지?"를 자연스럽게 따라감 |
| 4 | 대체로 잘 읽히나 1곳에서 속도가 느려짐 |
| 3 | 2~3곳에서 멈칫하거나 재독 필요 |
| 2 | 의구심이 자주 발생, 독자가 능동적으로 해석해야 함 |
| 1 | 읽는 중 길을 잃는 느낌, 전체 재독 필요 |

---

## 피드백 우선순위

| 티어 | 기준 | 피드백 활성화 조건 |
|------|------|-----------------|
| 1순위 | R2 Focus, R5 Structure | 항상 활성 |
| 2순위 | R3 Syntax, R4 Cohesion, R6 Depth | R2·R5 Resolved 후 활성 |
| 3순위 | R7 Flow | R2~R6 전체 Resolved 후 활성 |

**피드백 상한**: 세션당 최대 3개. 가장 낮은 점수 기준 우선 티어 순으로 선정.

## 레벨별 합격선

| 레벨 | 대상 | 합격선 |
|------|------|--------|
| Lv.1 | 중학생 | R1 Gate + 12/30 |
| Lv.2 | 고등학생 | R1 Gate + 15/30 |
| Lv.3 | 수능 논술 | R1 Gate + 18/30 |
| Lv.4 | 대학 리포트 / TOEFL | R1 Gate + 21/30 |
| Lv.5 | GRE AW | R1 Gate + 24/30 |
| Lv.6 | 실무 보고서 | R1 Gate + 26/30 |
| Lv.7 | 학술 논문 | R1 Gate + 28/30 |

---

## 레벨 자동 감지

글 제출 시 선언 레벨과 실제 수준이 다르면 재채점을 제안한다.

| 상황 | 처리 |
|------|------|
| 실제 수준 > 선언 레벨 | "실제 수준 Lv.N으로 재채점을 제안한다" + 재채점 결과 병기 |
| 실제 수준 < 선언 레벨 | 선언 레벨 기준 채점 유지, 부족한 부분 명시 |
| 일치 | 그대로 진행 |

**판단 기준**: R6 Depth 5점 + R5 Structure 4점 이상이면 선언 레벨보다 1~2단계 높은 수준으로 판단한다.

---

## 채점 출력 형식

```
[R1 Gate: Pass / Soft Fail / Hard Fail]
[모드: 일반 / Timed]
총점: N/30

피드백 (최대 3개, 우선순위 순):
1. [R? 기준명 - N점] 구체적 문제 지적.
   개선 방향 한 문장.
2. ...
3. ...

[합격 여부: 통과 (합격선 N점) / 미달 (N점 필요)]
[레벨 감지: 선언 Lv.N / 실제 추정 Lv.N — 재채점 제안 여부]
```
