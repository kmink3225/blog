# 논리적 글쓰기 취약점 트래커 — 이준현

> 이 파일은 **이준현의 writing 튜터링 세션 시작 시 필수 로드**한다.
> 세션 종료 후 아래 업데이트 프로토콜에 따라 agent가 직접 수정한다.

---

## Active 취약점

현재 반복 발생 중인 실수 패턴이다.

| 기준 | 언어 | 최근 발생 | 누적 횟수 | 비고 |
|------|------|----------|----------|------|
| R5 Structure | 통합 | 2026-04-27(세션#6) | 8 | 04-15(1~2점) → 04-20(2점) → 04-23#3(3점) → 04-23#4(4점 4칸 진입) → 04-24#5(3점 퇴행) → **04-27#6(3점 고착, e스포츠 인정)**. 제약 3(축 1개 의무) **명시 부과했음에도 3축 혼재로 악화** — Thesis "사회 문화 측" 선언 → 문단 1 셋째 줄 "산업 규모도 동등하다"(경제 축) → 문단 4 "제도적인 측면이 활발히 연구"(제도 축). **3개 축이 한 글 안에 공존**. 카테고리 명명은 성공(LOL·스타크래프트·롤드컵·SK·KT·아시안게임 등 6+개) — 사용자가 "카테고리 많이 쓰기" 지시는 수용했으나 "한 축 안에서만" 조건은 흘려들었음. **진단**: 카테고리 명명과 축 수렴은 분리된 능력. 카테고리 폭은 늘었지만 축 수렴은 후퇴. 연속 성공 카운트: **0/3 유지**. 다음 세션은 R5 자체 검출 의무 — "본문 명사 중 선언한 축에 속하지 않는 것 5개 적기, 5개 미만이면 통과" |
| R4 Cohesion (문단 전환 표면적) | 통합 | 2026-04-27(세션#6) | 5 | 04-13·04-14·04-15 3점 → 04-23#3 3점 → 04-23#4 4점(브리지 최초) → 04-24#5 4점 유지 → **04-27#6 3점 퇴행**. 브리지 시도 흔적은 있음 — "또한 e스포츠에 대한 위상이 ... 올라와 있을 **뿐만 아니라**" — 그러나 이는 한 문단 안 단문 연결이지 **문단 간 브리지가 아님**. 문단 1→2, 2→3, 3→4 모두 "또한"·"이러한 사례를 보듯이"·"게임에 대한 대중들의 인지도는" 식 표면적 점프. 지난 세션 검증된 "뿐만 아니라" 패턴이 문단 내 연결로 **잘못 적용** — 사용자는 도구는 익혔으나 적용 위치를 혼동. 연속 성공 카운트: **2/3 → 0/3 리셋**. 다음 세션은 "문단 첫 문장에 직전 문단 핵심어 1개 의무 포함" 명시 부과 |
| R8 Paraphrasing (핵심 어휘 반복 + 결론 재진술 부재) | 통합 | 2026-04-27(세션#6) | 6 | 04-13·04-14·04-15 3점 → 04-23#3 3점 → 04-23#4 4점(개념 층위 치환 최초) → 04-24#5 3점 → **04-27#6 3점 고착**. "동등하다/동일하다/비슷해졌다" 동의어 변주는 있으나 층위 이동 아님. "스포츠/스포츠 종목" 단순 반복. "전세계" 3회 반복. "대중·사람들·대중들" 변주. 핵심 주장 재진술 부재. **외부 제약 명시 안 했음 — 04-23#4의 개념 층위 치환은 그 세션에서만 통하던 일회성 성과로 확정**. 연속 성공 카운트: **0/3 유지**. 다음 세션 명시 부과 보류(R5·R3·R4 부담 가중) — R5·R3 안정화 후 재부과 |
| R3 Syntax (오탈자·띄어쓰기·장문 구조) | 통합 | 2026-04-27(세션#6) | 6 | 04-14 3점 → 04-15·04-23#3·04-23#4·04-24#5 2점 → **04-27#6 2점 고착(6세션 연속)**. 체크 문장 `[통독 완료 — 수정 8건 처리]` **형식적으로 부착 성공**. 그러나 본문 잔존 오류가 잡힌 8건보다 많음 — "**왠만한**"(오타, 소리내면 즉시 걸림) + 띄어쓰기 15건 이상(투자규모·티켓 가격등·롤드컵 이라는·수백만명의·LOL종목을·서포트 하는·대기업 들이며·100억의 수준·10~20 만원대·스포츠 보다·**할수 있다 2회**·인식개선·좀더). **새로운 진단**: 통독 자체는 일어났을 가능성(8건 잡음) — 그러나 **검출 능력 자체가 부족**. 단순 통독으로는 검출되지 않는 패턴 존재. 능력 부족 + 작업흐름 저항 **이중 문제**로 재진단. 다음 세션 처방 강화: "통독 + 4항목 고정 체크리스트"(할수있다 / 들 띄어쓰기 / 하는 띄어쓰기 / 외래어+한자 합성어). 체크 문장 형식: `[통독 완료 — 4항목 점검 — 수정 X건]` |
| R10 Rhetoric (Hook·수사 장치 부재) | 통합 | 2026-04-27(세션#6) | 5 | 04-14 2점 → 04-23#3 2점 → 04-23#4 3점 → 04-24#5 3점 → **04-27#6 2점 퇴행**. Hook 부재 + 강조·대조 장치 없음. 지난 두 세션 3점은 "첫번째/두번째 이유" 구조 나열에 의한 약한 강조였으나 이번 세션은 그것조차 사라짐. 3순위 기준이라 피드백 억제 상태 유지. R5·R3·R8 Resolved 후 공식 피드백 전환 |
| R2 Focus (Thesis 내부 모호성 / 직후 흩어짐) | 통합 | 2026-04-27(세션#6) | 3 | 04-23#3 4점 → 04-23#4 5점 → 04-24#5 5점 → **04-27#6 3점 퇴행**. Thesis 한 문장 압축 자체는 유지("e스포츠는 전통적인 스포츠와 사회 문화 측에서 동등하게 인정받아야 한다" — 대상·비교·축 3변수 박힘) — 압축 기법은 살아있음. 그러나 **둘째·셋째 문장이 즉시 다른 축 도입**("기업의 투자규모 및 티켓 가격등 산업 규모도 동등하다") — 한 문장 압축이 글 전체를 지배 못함. **신규 진단**: Thesis 압축은 내재화됐으나 **Thesis 효력 유지**(축 키워드를 본문이 받아내는 폐쇄 구조)는 미내재화. 연속 성공 카운트: **2/3 → 0/3 리셋**. 다음 세션 처방: "Thesis 첫 문장의 축 키워드를 둘째 문장에 1회 의무 재출현" — 폐쇄 구조 강제 |
| R11 Diction (잉여 수식·부정확 어휘) | 통합 | 2026-04-27(세션#6) | 4 | 04-23#3 2점 → 04-23#4 3점 → 04-24#5 3점 → **04-27#6 2점 퇴행**. "왠만한"(오타·R3 중첩), "서포트 하는"(외래어 어색), "꾸준히 증가하고 있으며", "활발히 연구되어야 한다"(잉여), "할수 있다"(R3 중첩) 다수. R3 오류와 R11 잉여가 같은 문장에서 동시 발생 — 통독 검출 4항목 체크리스트(R3 처방)가 적용되면 R11도 부분 개선 예상. 3순위 기준 억제 상태 유지 |

## 튜터 측 교정 메모 (2026-04-27 세션#6, "e스포츠 동등 인정")

### 세션 개요

총점 **28/50** (합격선 38에서 10점 부족). 직전 세션(#5) 34점에서 **-6점 추가 퇴행**. **두 세션 연속 퇴행** — 트래커 기록상 첫 사례. 04-23#4 정점(37점) 대비 -9점.

### 핵심 진단 — "내재화 평가의 단기 불안정성"

지난 세션(#5) 평가에서 R2·R4를 "내재화 중(2/3)"으로 판정했으나, 이번 세션 두 항목 모두 3점으로 퇴행하며 **연속 카운트 리셋**. 두 세션의 4~5점 연속이 곧바로 내재화를 의미하지 않음을 확인.

| 기준 | #4 | #5 | #6 | 카운트 변화 |
|------|----|----|----|-----------|
| R2 Focus | 5 | 5 | **3** | 2/3 → 0/3 리셋 |
| R4 Cohesion | 4 | 4 | **3** | 2/3 → 0/3 리셋 |
| R5 Structure | 4 | 3 | **3** | 0/3 유지 |
| R8 Paraphrasing | 4 | 3 | **3** | 0/3 유지 |
| R3 Syntax | 2 | 2 | **2** | 6세션 연속 2점 |
| R10 Rhetoric | 3 | 3 | **2** | 신규 퇴행 |
| R11 Diction | 3 | 3 | **2** | 신규 퇴행 |

**결론**: Resolved 판정 기준(3회 연속 4점)이 보수적인 데에는 이유가 있다. 2회 성공 후 한 번이라도 무너지면 그 성공은 외부 제약·주제 적합성·우연 중 하나에 의존했음을 시사. 정직한 카운트 운용이 진단 정확도를 보장.

### R2 신규 진단 — "압축은 내재화, 효력 유지는 미내재화"

이번 세션은 Thesis 압축 자체는 살아있었다 — "e스포츠는 전통적인 스포츠와 사회 문화 측에서 동등하게 인정받아야 한다"는 대상·비교·축 3변수가 한 문장에 박혔다. 그러나 **둘째·셋째 문장이 즉시 다른 축을 끌어옴**으로써 Thesis가 글 전체를 지배하지 못했다.

이는 R2의 평가 차원이 두 개임을 시사:

1. **압축 차원** — Thesis를 한 문장에 변수 3개 박을 수 있는가 (이준현: 내재화 ○)
2. **효력 차원** — Thesis가 본문을 폐쇄 구조로 가두는가 (이준현: 미내재화)

기존 트래커는 두 차원을 구분하지 않고 "R2 Focus" 한 행으로 운용했음. 다음 세션은 **둘째 차원에 직접 작용하는 처방**을 부과한다 — "Thesis 첫 문장의 축 키워드를 둘째 문장에 1회 재출현시켜라"는 단순한 강제가 폐쇄 구조 형성을 시작점으로 삼는다.

### R5 신규 진단 — "카테고리 명명과 축 수렴은 분리된 능력"

이번 세션 카테고리 명명은 풍부했다 — LOL, 스타크래프트, 롤드컵, SK, KT, 아시안게임, 100억 연봉, 10~20만원 티켓 등 6개 이상. **제약 2(카테고리 3개 의무)는 성공적 수용**.

그러나 **제약 3(축 1개 의무)은 정반대 방향으로 이탈** — 사회문화 + 산업/경제 + 제도 3축 혼재로 #5(2축)보다 악화. 사용자는 "구체화"를 "다양성"으로 잘못 해석한 것으로 추정 — 더 많은 종류의 카테고리를 들수록 좋다고 받아들이고, 그것이 한 축 안에서여야 한다는 조건은 흘려들었음.

**다음 세션 처방**: 사용자 자신이 검출하는 의무 부과 — "본문 명사 중 선언한 축에 속하지 않는 것 5개를 적어라. 5개 미만이면 통과, 5개 이상이면 본문 재작성." 이는 카테고리 명명과 축 수렴이 **별개 작업**임을 행동으로 학습시킨다.

### R3 신규 진단 — "능력 부족 + 작업흐름 저항 이중 문제"

체크 문장 `[통독 완료 — 수정 8건 처리]`가 형식적으로 부착되었다 — **#5의 "통독 자체 미실행" 단계는 통과**. 8건 수정한 흔적도 있다(추정). 그러나 본문 잔존 오류가 잡힌 8건보다 더 많은 15+건.

이는 #5 진단("작업흐름 저항")만으로 설명되지 않는 새 데이터다:

- #5: 통독 미실행 — 작업흐름 저항
- #6: 통독 실행, 8건 수정 — 그러나 검출 능력 부족

**진단 갱신**: R3는 단일 원인이 아니라 (a) 작업흐름 저항 + (b) 검출 능력 부족 이중 문제. 체크 문장 도입은 (a)를 부분 해결했고, 다음 단계는 (b)를 공략한다.

**처방**: 통독에 **고정 체크리스트 4항목**을 결합 — 검출 패턴을 외부 도구로 제공해 검출 능력 부족을 보완.

1. "할수 있다 / 될수 있다" 패턴 검사 (이번 세션 2회 출현)
2. "OO들 / OO 들" 띄어쓰기 검사 ("대기업 들이며" 패턴)
3. "OO하는 / OO 하는" 띄어쓰기 검사 ("서포트 하는" 패턴)
4. 외래어+한자 합성어 띄어쓰기 검사 ("LOL종목" 패턴)

체크 문장 형식 확장: `[통독 완료 — 4항목 점검 — 수정 X건]`

### R4 새 데이터 — "도구는 익혔으나 적용 위치 혼동"

이번 세션 본문에 "**뿐만 아니라**" 표현이 등장했다 — 04-23#4에서 검증된 브리지 패턴이다. 그러나 등장 위치가 **문단 3 내부 단문 연결**("e스포츠에 대한 위상이 ... 올라와 있을 뿐만 아니라, 대기업의 주요 광고 수단으로")로, **문단 간 브리지가 아님**.

도구 자체는 사용자 도구함에 들어왔으나 사용 위치를 혼동. 다음 세션 처방: "문단 첫 문장에 직전 문단 핵심어 1개 의무 포함" — 위치를 강제해 도구의 정확한 사용처를 학습.

### 차기 세션 예상

## 업데이트 프로토콜

### 언제 업데이트하는가

- `/save` 트리거 또는 세션 종료 시
- 3편 미만 세션은 노이즈로 간주, 업데이트하지 않는다

### 어떻게 업데이트하는가

**피드백이 발생한 경우 (Active 업데이트):**

1. 해당 기준이 이미 Active 테이블에 있으면 — 최근 발생 날짜 갱신, 누적 횟수 +1
2. 없는 새 패턴이면 — Active 테이블에 행 추가
3. `last_updated` 날짜 갱신

**개선된 경우 (Resolved 이동 조건):**

- Active 기준이 **3회 연속 4점 이상**이면 Resolved로 이동
- Resolved 이동 시: Active에서 행 삭제 → Resolved 테이블에 추가

### 판단 기준

| 상황 | 처리 |
|------|------|
| 해당 기준 4점 이상 | 연속 성공 카운트 +1 |
| 3점 이하 | 연속 성공 카운트 리셋 |
| 3회 연속 4점 이상 | Resolved로 이동 |
| 새로운 약점 발견 | Active에 추가 |

## 피드백 억제 규칙

| 티어 | 기준 | 활성화 조건 |
|------|------|-----------|
| 1순위 | R2 Focus, R5 Structure | 항상 피드백 |
| 2순위 | R3 Syntax, R4 Cohesion, R6 Depth, R8 Paraphrasing, R9 Universality | R2·R5 모두 Resolved 후 활성 |
| 3순위 | R7 Flow, R10 Rhetoric, R11 Diction | R2~R6, R8~R9 전체 Resolved 후 활성 |

R2·R5가 Active인 동안은 2순위·3순위 피드백을 억제한다.
