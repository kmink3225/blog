# 논리적 글쓰기 취약점 트래커 — 이준현

> 이 파일은 **이준현의 writing 튜터링 세션 시작 시 필수 로드**한다.
> 세션 종료 후 아래 업데이트 프로토콜에 따라 agent가 직접 수정한다.

## Active 취약점

현재 반복 발생 중인 실수 패턴이다.

| 기준 | 언어 | 최근 발생 | 누적 횟수 | 비고 |
|------|------|----------|----------|------|
| R5 Structure | 통합 | 2026-04-23 | 5 | 2026-04-15(1~2점 김구·안중근 사실 오류 + 기능적 대응 실패) → 04-20(2점, "과거 세대 책임" 주제에서 Reasoning 과포괄로 Example 경로 봉쇄) → **04-23(3점, Lv.2 미성년 신상공개)**. 이번 세션에 고유 인물 사건(조두순·N번방) 회피는 성과(+1점 회복). 그러나 Example이 "실제로 범죄율이 낮아졌다" "재범률이 현저히 떨어졌다" 같은 **출처 없는 일반화 진술**로 대체됨. 이동 궤적: "없음 → 아는 이름 나열 → 통계 뉘앙스 일반화". 다음 칸은 **제도 층위 구체화**(소년법 신상공개 예외 규정 작동 구조, 성인 전자발찌 제도 도입 전후 비교 구조). 숫자 없이도 제도 로직만으로 Claim 증명하는 훈련 필요. 추가로 Reasoning 축 분산 재발(공익·재범·관리감찰·응보·반성 5축 혼재) — Lv.2에서는 한 축만 사다리 끝까지 내리는 연습 강제 |

| R4 Cohesion (문단 전환 표면적) | 통합 | 2026-04-23 | 4 | 04-13 3점 → 04-14 3점 → 04-15 3점 → **04-23 3점 재발** (04-20 데이터 불명, 4회 누적). "공공의 안전 뿐만 아니라, 신상공개는…" 같은 단순 연결어 여전. 앞 문단 마지막 문장에서 다음 문단을 예고하는 브리지 문장 **4세션째 시도 없음**. R2·R5 Active로 공식 피드백은 억제됐으나 패턴 고착 매우 심화. R5 Resolved 즉시 최우선 교정 대기 |
| R8 Paraphrasing (핵심 어휘 반복 + 결론 재진술 부재) | 통합 | 2026-04-23 | 4 | 04-13·04-14·04-15 3점 → **04-23 3점 재발** (누적 4회). 이번엔 "신상 공개" "필요하다" "공공 안전"이 거의 동일 표현으로 반복. 결론 문단 "신상공개로 평생 사람들의 기억속에 남아 뉘우치면서 살아야" 외에는 서두·본문 어휘 그대로 재사용. **개념 층위 이동(신상 공개 → 사회적 제재 낙인 / 공동체 기억 장치)** 훈련 여전히 체화 안 됨 |
| R3 Syntax (오탈자·띄어쓰기·장문 구조) | 통합 | 2026-04-23 | 3 | 04-14 3점 → 04-15 2점 → **04-23 2점 악화 지속**. 이번 세션 오류: "신상은 공개는"(조사 중복), "있으며."(마침표 위치), "어지렵인"(→어지럽힌), "재범률을 막기"(대상 오용, →재범 억제), "인지를 하기에"(→인지하기에), "공개 라는"(띄어쓰기). 문장 구조도 단순 반복 + "생각된다" 피동 약결말. 제출 전 통독 루틴 부재 **3세션 연속 방치**. | R2 Focus (Thesis 내부 모호성) | 통합 | 2026-04-23 | 1 | **신규 약점 — 자가점검 3 실패**. 04-23 세션 서두 첫 문장("신상 공개는 필요하다" = 전면 공개 찬성)과 세 번째 문장("강력범죄의 경우 공개가 필요하다" = 조건부 공개 찬성)이 **서두 안에서 이미 다른 Thesis**. 본문은 후자로 전개되어 R2 4점 방어는 됐으나, 서두 자체에서 입장 좁히기 실패. 다음 세션 자가점검 3을 "Thesis 한 문장에 범위·대상·조건 모두 못 박기" 수준으로 강화 필요 |
| R11 Diction (잉여 수식·부정확 어휘) | 통합 | 2026-04-23 | 1 | **신규 기록 — 억제 상태**. 04-23 세션 2점. "공익의 안전을 어지렵인 죄를 알아야 한다"(부정확 어휘 + 비문), "생각된다"(피동 약결말), "신상공개 라는 무거운 처벌"(잉여 수식) 등. 3순위 기준이라 공식 피드백 억제 중이나 R10과 함께 Lv.3 진입 전 해소 필요 패턴 |

## 튜터 측 교정 메모 (2026-04-23)

### 이번 세션 진단 — R5 Example의 "3번째 칸" 이동

R5 Example은 한 세션마다 한 칸씩 움직이는 궤적을 보인다.

- **1칸 (04-15)**: Example이 아예 없거나 고유 인물 이름 나열("김구·안중근")
- **2칸 (04-20)**: Reasoning이 너무 포괄적이라 Example로 가는 경로가 **봉쇄**됨
- **3칸 (04-23)**: Example을 제도·통계 일반화 진술로 **대체** ("실제로 범죄율이 낮아졌다")
- **4칸 (목표)**: 제도 로직 자체로 Claim 증명 (소년법 신상공개 예외 규정의 작동 구조 + 특정 범죄소년 연령 구간에서 해당 규정이 어떻게 재범 의사결정을 제약하는지 단계 추론)

**핵심 인사이트**: 사용자는 "구체 사건 이름"과 "제도 층위" 사이에 **"통계 뉘앙스 진술"**이라는 중간 지대를 만들어 피신 중이다. 이 지대는 출처가 없어 검증 불가능하고, Example의 기능(Claim 증명)을 수행하지 않는다. 다음 세션은 "통계 진술 금지 — 제도 구조만으로 증명하라"를 명시적 제약으로 제시한다.

### 이번 세션에서 관찰된 신규 패턴 — "Thesis 내부 좁히기" 실패

자가점검 3을 출제 시 강조했음에도 **서두 한 문단 안에서** Thesis 폭이 좁아지는 현상이 발생했다.

- 서두 첫 문장: 범위 무제한("신상 공개는 필요하다")
- 서두 세 번째 문장: 강력범죄 한정("강력범죄의 경우 필요하다")

이는 사용자가 Thesis를 **한 문장에 압축하는 훈련**이 부재함을 시사한다. 두 문장에 걸쳐 Thesis를 "펼쳐서" 쓰는 습관이다. 다음 세션부터는 **"Thesis는 첫 문장 하나, 그 문장 안에 대상·범위·조건 모두 포함"** 제약을 추가한다.

### Example 훈련 전용 미니 프롬프트 (차기 세션용 준비)

사용자가 Example "3칸 → 4칸" 이동을 체화하도록 다음 세션에 아래 제약을 추가 검토:

> "본 Example 단계에서 숫자·통계·일반화 진술 사용 금지. 제도 조문, 연령 구간, 범죄 유형 분류, 절차 단계 중 **3가지 이상의 제도 층위 단위**를 Example 문단에 직접 명명하라."

이 제약이 통과되면 R5가 4~5점 영역 진입 가능.

### R5 피드백 적용 순서 (현행 유지, 04-15에서 정립)

1. **각 문장의 추상도 매핑** (고/중/저)
2. **축별 구체화 사다리 추적** — 주체·행위·의미 등 평행 축이 단계적으로 내려가는가
3. **Reasoning → Example 경로 시뮬레이션** — 튜터가 직접 "이 Reasoning에서 어떤 Example이 도출되는가?"를 머릿속에서 써본다
4. 경로 봉쇄 시 → **Example 교체가 아니라 Reasoning 문장 재설계를 요구**
5. 이후에 Example-Claim 기능적 대응 점검
6. 마지막에 사실 정확성

## 튜터 측 교정 메모 (2026-04-15)

### 1차 교정 — 사실 정확성 ≠ R5의 핵심
오늘 세션에서 튜터(Claude)가 김구·K-1 소총의 사실 오류를 1순위 피드백으로 제시했으나, 사용자가 교정했다 — **진짜 축은 "왜 하필 이 예시가 이 Claim을 증명하는가"라는 기능적 필연성**이다. 사실을 바로잡아도 논리 연결이 없으면 같은 글이 된다.

### 2차 교정 — R-C-E는 단계가 아니라 "구체화 사다리"
사용자가 한 단계 더 깊은 분석틀을 제시: **Claim/Reasoning/Example은 "있냐/없냐"의 체크리스트가 아니라, 평행한 축들(주체·행위·의미)이 각각 추상에서 구체로 내려가는 사다리**다.

기대수준 예시(유관순)의 3축 구체화:
- 주체 축: 독립운동가 → 영웅 → 유관순
- 행위 축: 기리는 것 → 공동체가 기억 → 집단 가치 상징
- 의미 축: 국가 정체성 근간 → 지향 가치 → 공동체 신념 → 현재 정체성 선언

유관순은 "아는 이름"이라 등장한 게 아니라 **세 축 사다리의 바닥에 논리적으로 착지**한 결과. 이준현의 Reasoning 문장("독립운동가들은 자기 희생 및 교육을 통해 애국심을 강조했고 현재까지 유지된다")은 **너무 포괄적이어서 구체화 경로가 봉쇄**됨 → 사다리를 못 타고 "김구/안중근"으로 점프. 증상은 사실 오류지만 원인은 **Reasoning 문장 설계 실패**.

### R5 피드백 적용 순서 (개정판)

1. **각 문장의 추상도 매핑** (고/중/저)
2. **축별 구체화 사다리 추적** — 주체·행위·의미 등 평행 축이 단계적으로 내려가는가
3. **Reasoning → Example 경로 시뮬레이션** — 튜터가 직접 "이 Reasoning에서 어떤 Example이 도출되는가?"를 머릿속에서 써본다
4. 경로 봉쇄 시 → **Example 교체가 아니라 Reasoning 문장 재설계를 요구**
5. 이후에 Example-Claim 기능적 대응 점검
6. 마지막에 사실 정확성

**핵심 원칙: 증상(사실 오류, Example 부재)이 아니라 원인(Reasoning 문장의 디벨롭 가능성)을 진단한다.**

## 업데이트 프로토콜

### 언제 업데이트하는가

- `/save` 트리거 또는 세션 종료 시
- 3편 미만 세션은 노이즈로 간주, 업데이트하지 않는다

### 어떻게 업데이트하는가

**피드백이 발생한 경우 (Active 업데이트):**

1. 해당 기준이 이미 Active 테이블에 있으면 — 최근 발생 날짜 갱신, 누적 횟수 +1
2. 없는 새 패턴이면 — Active 테이블에 행 추가
3. `last_updated` 날짜 갱신

**개선된 경우 (Resolved 이동 조건):**

- Active 기준이 **3회 연속 4점 이상**이면 Resolved로 이동
- Resolved 이동 시: Active에서 행 삭제 → Resolved 테이블에 추가

### 판단 기준

| 상황 | 처리 |
|------|------|
| 해당 기준 4점 이상 | 연속 성공 카운트 +1 |
| 3점 이하 | 연속 성공 카운트 리셋 |
| 3회 연속 4점 이상 | Resolved로 이동 |
| 새로운 약점 발견 | Active에 추가 |

## 피드백 억제 규칙

| 티어 | 기준 | 활성화 조건 |
|------|------|-----------|
| 1순위 | R2 Focus, R5 Structure | 항상 피드백 |
| 2순위 | R3 Syntax, R4 Cohesion, R6 Depth, R8 Paraphrasing, R9 Universality | R2·R5 모두 Resolved 후 활성 |
| 3순위 | R7 Flow, R10 Rhetoric, R11 Diction | R2~R6, R8~R9 전체 Resolved 후 활성 |

R2·R5가 Active인 동안은 2순위·3순위 피드백을 억제한다.
