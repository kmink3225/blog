# 논리적 글쓰기 취약점 트래커 — 이준현

> 이 파일은 **이준현의 writing 튜터링 세션 시작 시 필수 로드**한다.
> 세션 종료 후 아래 업데이트 프로토콜에 따라 agent가 직접 수정한다.

## Active 취약점

현재 반복 발생 중인 실수 패턴이다.

| 기준 | 언어 | 최근 발생 | 누적 횟수 | 비고 |
|------|------|----------|----------|------|
| R5 Structure | 통합 | 2026-04-14 | 2 | (초기 진단 → 가짜뉴스 주제 세션 3점 재발). 최초 진단은 "논리 비약(Reasoning 누락)" 관점이었으나 2026-04-14 세션에서는 **Example 단계 부재**로 같은 R5 3점 기록 — Claim → Reasoning → Example → 마무리 4단계 중 "구체 사례" 단계 체화가 다음 과제. Reasoning은 등장하기 시작 |
| R6 Depth (논거 develop 약함) | 통합 | 2026-04-14 | 2 | 서로 다른 관점의 논거 2개(의도·신뢰)까지 제시하는 수준 도달(4점). 다음 단계는 각 논거에 **반례 고려 + 구체 증거 + 예상 반론 선제 방어**를 붙이는 develop. 연속 성공 카운트: **1 / 3** (Resolved까지 2회 남음) |
| R4 Cohesion (문단 전환 표면적) | 통합 | 2026-04-14 | 2 | 어제(04-13) 3점 → 오늘(04-14) 3점 **연속 재발**. "또한", "결론적으로" 단순 연결어 수준에 머무름. 앞 문단 마지막 문장에서 다음 문단을 예고하는 브리지 문장 체화 필요. 패턴 고착 경향, 우선 교정 대상 |
| R8 Paraphrasing (핵심 어휘 반복 + 결론 재진술 부재) | 통합 | 2026-04-14 | 2 | 어제(04-13) 3점 → 오늘(04-14) 3점 **연속 재발**. 핵심 어휘 그대로 반복("가짜뉴스/뉴스/매체/책임"), 결론 재진술이 주체 전가("기자·언론사에게")에 그침. 같은 개념을 다른 말로 바꾸는 변주 훈련 필요 |
| R3 Syntax (오탈자·띄어쓰기·장문 구조) | 통합 | 2026-04-14 | 1 | 2026-04-14 세션 3점. 오탈자·띄어쓰기 다수("개시물/게시물", "당사자 이며", "볼수", "생각이든다", "뉴스 라는"), 복합 문장이 지나치게 길어 구조 꼬임(본론 2의 한 문장). 제출 전 교정 습관 + 긴 문장 분할 훈련 필요 |
| R10 Rhetoric (Hook·수사 장치 부재) | 통합 | 2026-04-14 | 1 | 2026-04-14 세션 2점(가장 낮음). Hook(질문·사례·대조) 없이 바로 주장 진입, 본문에 강조·대조·반복 등 수사 장치 거의 없음. 3순위 기준이라 현재 억제 대상이나 기록은 남김 — R2·R5·R4·R6·R8 Resolved 후 공식 피드백 대상 |

## 업데이트 프로토콜

### 언제 업데이트하는가

- `/save` 트리거 또는 세션 종료 시
- 3편 미만 세션은 노이즈로 간주, 업데이트하지 않는다

### 어떻게 업데이트하는가

**피드백이 발생한 경우 (Active 업데이트):**

1. 해당 기준이 이미 Active 테이블에 있으면 — 최근 발생 날짜 갱신, 누적 횟수 +1
2. 없는 새 패턴이면 — Active 테이블에 행 추가
3. `last_updated` 날짜 갱신

**개선된 경우 (Resolved 이동 조건):**

- Active 기준이 **3회 연속 4점 이상**이면 Resolved로 이동
- Resolved 이동 시: Active에서 행 삭제 → Resolved 테이블에 추가

### 판단 기준

| 상황 | 처리 |
|------|------|
| 해당 기준 4점 이상 | 연속 성공 카운트 +1 |
| 3점 이하 | 연속 성공 카운트 리셋 |
| 3회 연속 4점 이상 | Resolved로 이동 |
| 새로운 약점 발견 | Active에 추가 |

## 피드백 억제 규칙

| 티어 | 기준 | 활성화 조건 |
|------|------|-----------|
| 1순위 | R2 Focus, R5 Structure | 항상 피드백 |
| 2순위 | R3 Syntax, R4 Cohesion, R6 Depth, R8 Paraphrasing, R9 Universality | R2·R5 모두 Resolved 후 활성 |
| 3순위 | R7 Flow, R10 Rhetoric, R11 Diction | R2~R6, R8~R9 전체 Resolved 후 활성 |

R2·R5가 Active인 동안은 2순위·3순위 피드백을 억제한다.
