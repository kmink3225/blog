# 논리적 글쓰기 취약점 트래커 — 이준현

> 이 파일은 **이준현의 writing 튜터링 세션 시작 시 필수 로드**한다.
> 세션 종료 후 아래 업데이트 프로토콜에 따라 agent가 직접 수정한다.

## Active 취약점

현재 반복 발생 중인 실수 패턴이다.

| 기준 | 언어 | 최근 발생 | 누적 횟수 | 비고 |
|------|------|----------|----------|------|
| R5 Structure | 통합 | 2026-04-24(세션#5) | 7 | 04-15(1~2점) → 04-20(2점) → 04-23#3(3점) → 04-23#4(**4점 4칸 진입 성공**) → **04-24#5(3점 퇴행, 부모 SNS 감시)**. **외부 제약 완화 시 즉시 기본 습관 복귀 확인** — 지난 세션 금지어("실제로 ~", "경향이 나타났다")를 이번 세션은 부드럽게만 권고했더니 **두 표현 모두 재등장**("실제 이성친구간의 카카오톡… 사건이 있다", "실제로… 경향이 나타났다"). Example 이동 궤적이 "구체 카테고리 → 출처 불명 서사"로 **한 칸 퇴보**. 추가 문제: (1) 축 2개 혼재 재발(프라이버시 + 신뢰), (2) 카테고리 명명 부재(메시지 유형·감시 방식·대응 행동 등 구체 층위 미활용). **진단**: 04-23#4의 성과는 외부 제약 의존형 — 내재화 미완. 연속 성공 카운트: **0/3 리셋**. 다음 세션은 금지어 목록 명시 부과 + Example 카테고리 3개 이상 의무 |
| R4 Cohesion (문단 전환 표면적) | 통합 | 2026-04-24(세션#5) | 4 | 04-13·04-14·04-15 3점 → 04-23#3 3점 → 04-23#4 4점(브리지 최초 등장) → **04-24#5 4점 유지**. 문단 2→3 "프라이버시를 지켜주지 못하는 문제 **뿐만 아니라**, 부모 자식 서로간에 믿음 관계 형성에도 안좋은 영향을 끼친다" — 지난 세션 브리지 패턴 **재현 성공**. 문단 1→2는 Thesis의 "첫번째/두번째" 예고가 보완해주고, 3→4는 여전히 갑작스러움. 한 축 4세션 연속 4점 진입 안 했으나 재현성 확인. 연속 성공 카운트: **2/3** (Resolved까지 1회 남음) |
| R8 Paraphrasing (핵심 어휘 반복 + 결론 재진술 부재) | 통합 | 2026-04-24(세션#5) | 5 | 04-13·04-14·04-15 3점 → 04-23#3 3점 → 04-23#4 4점(개념 층위 치환 최초) → **04-24#5 3점 퇴행**. 지난 세션 "신체적 약점→신체 결함→취약 부분" 수준 층위 이동 소실. 이번 세션은 "감시"·"침해"·"카카오톡" 단순 반복. "프라이버시/사생활/개인정보"는 변주 있으나 층위 이동이라기보다 동의어 교체 수준. **외부 제약 완화 시 즉시 기본 습관 복귀**(R5와 동일 패턴). 연속 성공 카운트: **0/3 리셋**. 다음 세션 "핵심 어휘 1회 이상 개념 층위 치환" 명시 부과 필요 |
| R3 Syntax (오탈자·띄어쓰기·장문 구조) | 통합 | 2026-04-24(세션#5) | 5 | 04-14 3점 → 04-15 2점 → 04-23#3 2점 → 04-23#4 2점 → **04-24#5 2점 고착**. **5세션 연속 2점 방치**. 이번 세션이 특히 심각 — **의무 제약(제출 전 3분 소리내어 통독)을 명시적으로 부과했음에도 불구하고** "**긍적적인**"(오타, 소리내면 혀에서 걸림) + 띄어쓰기 10건 이상(첫번째·두번째·중고등 학생·특정인 끼리만·이성친구간의·몇달간·안좋은·서로간에·악화 시키는·침해 당했다는) + 조사·어미 오류 다수. **통독 루틴 실행 자체가 의심되는 상황**. 능력 문제가 아니라 **작업흐름 저항** 문제로 판명. 다음 세션 제약 2단계 강화: 제출 직전 "통독 완료, 수정 X건 처리" **체크 문장 의무 부착**. 체크 문장 없으면 채점 거부 |
| R10 Rhetoric (Hook·수사 장치 부재) | 통합 | 2026-04-24(세션#5) | 4 | 04-14 2점 → 04-23#3 2점 → 04-23#4 3점 → **04-24#5 3점 유지**. "첫번째/두번째 이유" 구조적 나열 외에는 Hook·강조·대조 장치 거의 없음. 3순위 기준이라 억제 상태 유지. R5·R3·R8 Resolved 후 공식 피드백 전환 |
| R2 Focus (Thesis 내부 모호성) | 통합 | 2026-04-24(세션#5) | 2 | 04-23#3 4점(신규 약점) → 04-23#4 5점 → **04-24#5 5점 유지**. Thesis 첫 문장 압축 기법이 **외부 제약 없이도 재현**됨 ("중고등학생 자녀 / 카카오톡 대화창 / 주기적 감시"로 대상·객체·방식 3변수 박힘). R5·R8과 달리 **R2는 내재화 중**. 연속 성공 카운트: **2/3** (Resolved까지 1회 남음) |
| R11 Diction (잉여 수식·부정확 어휘) | 통합 | 2026-04-24(세션#5) | 3 | 04-23#3 2점 → 04-23#4 3점 → **04-24#5 3점 유지**. "부모라는 권리로 공유를 강요한다면"(어색), "서로간에 믿음 관계 형성"(잉여), "관리해야 한다는 명목하에" 등 잉여 구문 남음. 3순위 기준 억제 상태 유지 |

## 튜터 측 교정 메모 (2026-04-24 세션#5, "부모 SNS 감시")

### 세션 개요

총점 **34/50** (합격선 38에서 4점 부족). 직전 세션(#4) 37점에서 **-3점 퇴행**. 5세션 연속 기록 중 **첫 퇴행**.

### 핵심 진단 — "외부 제약 의존 vs 내재화" 분리 확인

이번 세션이 드러낸 가장 중요한 사실: 이준현의 지난 세션(#4) 성과는 **외부 제약의 성공적 수용**이었지 **내재화**가 아니었다. 제약을 부드럽게 풀어주자 여러 지표가 동시 퇴행했다.

| 기준 | 04-23#4 | 04-24#5 | 내재화 여부 |
|------|--------|--------|------------|
| R2 Focus (Thesis 압축) | 5점 | 5점 | **내재화 ○** (외부 제약 없어도 재현) |
| R4 Cohesion (브리지 문장) | 4점 | 4점 | **내재화 ○** (재현 성공) |
| R5 Example 카테고리 | 4점 | 3점 | **외부 의존 ×** (금지어 명시 안 하자 재발) |
| R8 개념 층위 치환 | 4점 | 3점 | **외부 의존 ×** (명시 안 하자 단순 반복 복귀) |
| R5 축 하나 수렴 | 부분 성공 | 실패 | **외부 의존 ×** (2축 혼재 기본값 복귀) |
| R3 통독 루틴 | 미실행 | 미실행 (명시 부과했음에도) | **실행 자체 저항 중** |

**결론**: R2·R4는 이미 습관이 됐다. R5·R8·R3는 여전히 외부 제약에 100% 의존. 특히 R3는 **명시적으로 부과해도 실행되지 않는** 저항 단계다.

### R3의 특수 위상 — 능력이 아니라 작업흐름 저항

이번 세션은 "제출 전 3분 타이머 + 소리내어 통독"을 **의무 제약**으로 부과했다. 그럼에도:

- "긍적적인" 오타 (소리내면 즉시 걸림)
- 띄어쓰기 10건 이상 (눈으로라도 잡힘)
- "인지하고." 뒤 줄바꿈으로 문장 분절

이 결과는 두 가지 중 하나를 의미한다.

1. **통독을 건너뜀** — 작업흐름 변경 자체가 일어나지 않음
2. **통독을 했으나 눈으로 빠르게 훑음** — 지시의 핵심("소리내어")이 전달 안 됨

어느 쪽이든 **외부 지시만으로는 실행을 보장할 수 없다**. 다음 세션은 **실행 증거를 글 자체에 남기게** 한다.

### 다음 세션 처방 — 체크 문장 의무 + 제약 다시 명시

1. **R3 의무 강화**: 글 맨 끝에 `[통독 완료 — 수정 X건 처리]` 체크 문장 부착. 없으면 채점 거부. X가 0이면 제출 전 한 번 더 읽게 요구
2. **R5 Example 제약 재부과**: 금지어 목록 명시(`"실제로 ~"`, `"경향이 나타났다"`, `"대부분"`, `"많은 경우"`). 본문에 **구체 카테고리 3개 이상 명명** 의무
3. **R5 축 수렴 의무**: 하나만 선택, 다른 축은 단 한 번도 언급 불가
4. **유지된 성과는 제약에 포함하지 않음**: R2 Thesis 압축, R4 브리지는 이미 내재화 — 굳이 다시 지시할 필요 없음. 부담 분산

### 차기 세션 예상

제약 재부과 + 체크 문장 도입 시:
- 단기(1~2세션): 37~38점 회복, 합격선 통과 가능
- 중기(3~5세션): R5·R8·R3가 외부 제약 없이도 재현되면 **진짜 내재화** 달성
- 장기: 내재화 달성 후 Lv.3 승급 타진

### 학습 관찰 — 퇴행은 후퇴가 아니라 데이터

세션 #5 퇴행은 학습 실패가 아니라 **외부 의존/내재화를 구분해주는 검증 실험**이었다. 이 정보가 없었다면 "#4에서 이미 습관화됐다"고 오판해 다음 단계로 넘어갈 뻔했다. 퇴행이 보여준 구분선을 근거로 다음 처방이 정교해진다.

## 튜터 측 교정 메모 (2026-04-23 세션#4, "경쟁 약점 이용")

### 세션 개요

총점 **37/50** (합격선 38에서 1점 차 미달). 직전 세션(#3) 30점에서 **+7점 상승** — 트래커 기록상 최대 폭 개선. 다수 지표 동시 진전.

### 주요 성과 5건 동시 달성

1. **R5 Example "4칸" 진입**: "통계 일반화"→"구체 종목·전술 카테고리" 이동 완료. 3세션에 걸쳐 3칸 이동(없음→이름 나열→통계→카테고리)
2. **R2 Thesis 한 문장 압축 성공**: 예시 틀("X 경쟁에서 Y 약점을 Z 방식") 완전 수용, 서두 1문장에 3변수 박음
3. **R4 브리지 문장 최초 등장** (5세션 만): 문단 2→3 "뿐만 아니라" 연결어 + 앞 내용 요약 + 새 축 도입
4. **R8 개념 층위 치환 최초 등장** (5세션 만): "신체적 약점→신체 결함→신체구조→취약한 부분" 추상도 이동
5. **R6 Depth Resolved**: 연속 성공 3/3 달성, Active에서 이동

### 이번 세션 한계 2건

1. **R5 축 2개 혼재**: "효율성 + 규칙 합법성"으로 통합 시도. 제약 3의 "한 축만" 부분 위반. 5점 미달의 원인
2. **농구 Example의 Claim 기능 어긋남**: "키 큰 선수 고용"은 우리 팀 강화지 상대 약점 이용이 아님. Example 주어가 "상대 약점" vs "우리 강점" 중 어느 쪽에 있는지 점검 필요

### R3의 절대 고착 — 유일한 합격 저지 요인

R3 Syntax가 **4세션 연속 2점**. 이번에도 "스포즈"(오타), 띄어쓰기 4건, 조사 오류 등 7개 이상. 합격선 1점 차의 직접 원인. **R3가 2→4로만 가도 즉시 38점 돌파, 2→5면 40점 돌파**.

**진단**: 능력 문제가 아니라 **작업흐름 문제**. "쓰기→제출" 2단계 고정, "쓰기→통독→제출" 3단계 루틴 부재. 의지 기반 자가실행은 4세션 연속 실패했으므로 더 이상 의지에 맡기면 안 됨.

### 다음 세션 최우선 처방 — 장치 1개만 의무 부과

> **다음 세션 제약**: 제출 전 3분 타이머 설정 + **소리내어** 통독 1회. 타이머 끝까지 제출 불가.

이 하나만 체화되면 R3 + R4(브리지 검증) + R8(어휘 반복 감지) + R11(잉여 수식 감지)이 **동시에** 개선. 다른 제약은 덜어내 부담 분산.

차기 세션 예상: 통독 루틴 성공 시 **40~42점권 진입**, R3가 4점 돌파하면 Resolved 후보 진입.

## 튜터 측 교정 메모 (2026-04-23 세션#3, "미성년 신상공개")

### 이번 세션 진단 — R5 Example의 "3번째 칸" 이동

R5 Example은 한 세션마다 한 칸씩 움직이는 궤적을 보인다.

- **1칸 (04-15)**: Example이 아예 없거나 고유 인물 이름 나열("김구·안중근")
- **2칸 (04-20)**: Reasoning이 너무 포괄적이라 Example로 가는 경로가 **봉쇄**됨
- **3칸 (04-23)**: Example을 제도·통계 일반화 진술로 **대체** ("실제로 범죄율이 낮아졌다")
- **4칸 (목표)**: 제도 로직 자체로 Claim 증명 (소년법 신상공개 예외 규정의 작동 구조 + 특정 범죄소년 연령 구간에서 해당 규정이 어떻게 재범 의사결정을 제약하는지 단계 추론)

**핵심 인사이트**: 사용자는 "구체 사건 이름"과 "제도 층위" 사이에 **"통계 뉘앙스 진술"**이라는 중간 지대를 만들어 피신 중이다. 이 지대는 출처가 없어 검증 불가능하고, Example의 기능(Claim 증명)을 수행하지 않는다. 다음 세션은 "통계 진술 금지 — 제도 구조만으로 증명하라"를 명시적 제약으로 제시한다.

### 이번 세션에서 관찰된 신규 패턴 — "Thesis 내부 좁히기" 실패

자가점검 3을 출제 시 강조했음에도 **서두 한 문단 안에서** Thesis 폭이 좁아지는 현상이 발생했다.

- 서두 첫 문장: 범위 무제한("신상 공개는 필요하다")
- 서두 세 번째 문장: 강력범죄 한정("강력범죄의 경우 필요하다")

이는 사용자가 Thesis를 **한 문장에 압축하는 훈련**이 부재함을 시사한다. 두 문장에 걸쳐 Thesis를 "펼쳐서" 쓰는 습관이다. 다음 세션부터는 **"Thesis는 첫 문장 하나, 그 문장 안에 대상·범위·조건 모두 포함"** 제약을 추가한다.

### Example 훈련 전용 미니 프롬프트 (차기 세션용 준비)

사용자가 Example "3칸 → 4칸" 이동을 체화하도록 다음 세션에 아래 제약을 추가 검토:

> "본 Example 단계에서 숫자·통계·일반화 진술 사용 금지. 제도 조문, 연령 구간, 범죄 유형 분류, 절차 단계 중 **3가지 이상의 제도 층위 단위**를 Example 문단에 직접 명명하라."

이 제약이 통과되면 R5가 4~5점 영역 진입 가능.

### R5 피드백 적용 순서 (현행 유지, 04-15에서 정립)

1. **각 문장의 추상도 매핑** (고/중/저)
2. **축별 구체화 사다리 추적** — 주체·행위·의미 등 평행 축이 단계적으로 내려가는가
3. **Reasoning → Example 경로 시뮬레이션** — 튜터가 직접 "이 Reasoning에서 어떤 Example이 도출되는가?"를 머릿속에서 써본다
4. 경로 봉쇄 시 → **Example 교체가 아니라 Reasoning 문장 재설계를 요구**
5. 이후에 Example-Claim 기능적 대응 점검
6. 마지막에 사실 정확성

## 튜터 측 교정 메모 (2026-04-15)

### 1차 교정 — 사실 정확성 ≠ R5의 핵심
오늘 세션에서 튜터(Claude)가 김구·K-1 소총의 사실 오류를 1순위 피드백으로 제시했으나, 사용자가 교정했다 — **진짜 축은 "왜 하필 이 예시가 이 Claim을 증명하는가"라는 기능적 필연성**이다. 사실을 바로잡아도 논리 연결이 없으면 같은 글이 된다.

### 2차 교정 — R-C-E는 단계가 아니라 "구체화 사다리"
사용자가 한 단계 더 깊은 분석틀을 제시: **Claim/Reasoning/Example은 "있냐/없냐"의 체크리스트가 아니라, 평행한 축들(주체·행위·의미)이 각각 추상에서 구체로 내려가는 사다리**다.

기대수준 예시(유관순)의 3축 구체화:
- 주체 축: 독립운동가 → 영웅 → 유관순
- 행위 축: 기리는 것 → 공동체가 기억 → 집단 가치 상징
- 의미 축: 국가 정체성 근간 → 지향 가치 → 공동체 신념 → 현재 정체성 선언

유관순은 "아는 이름"이라 등장한 게 아니라 **세 축 사다리의 바닥에 논리적으로 착지**한 결과. 이준현의 Reasoning 문장("독립운동가들은 자기 희생 및 교육을 통해 애국심을 강조했고 현재까지 유지된다")은 **너무 포괄적이어서 구체화 경로가 봉쇄**됨 → 사다리를 못 타고 "김구/안중근"으로 점프. 증상은 사실 오류지만 원인은 **Reasoning 문장 설계 실패**.

### R5 피드백 적용 순서 (개정판)

1. **각 문장의 추상도 매핑** (고/중/저)
2. **축별 구체화 사다리 추적** — 주체·행위·의미 등 평행 축이 단계적으로 내려가는가
3. **Reasoning → Example 경로 시뮬레이션** — 튜터가 직접 "이 Reasoning에서 어떤 Example이 도출되는가?"를 머릿속에서 써본다
4. 경로 봉쇄 시 → **Example 교체가 아니라 Reasoning 문장 재설계를 요구**
5. 이후에 Example-Claim 기능적 대응 점검
6. 마지막에 사실 정확성

**핵심 원칙: 증상(사실 오류, Example 부재)이 아니라 원인(Reasoning 문장의 디벨롭 가능성)을 진단한다.**

## Resolved 취약점

3회 연속 4점 이상으로 극복 판정된 패턴이다. 이력 보존용으로 삭제하지 않는다.

| 기준 | 언어 | 해결 날짜 | 총 발생 횟수 | 비고 |
|------|------|----------|------------|------|
| R6 Depth (논거 develop 약함) | 통합 | 2026-04-23(세션#4) | 3 | 04-14 4점(첫 4점 달성) → 04-15 유보(R5 치명) → 04-23#3 4점(연속 2/3) → **04-23#4 4점(연속 3/3 달성)** → Resolved 이동. 안정 구간: 서로 다른 관점의 논거 2개 이상 제시 능력 확보(사회·제도 축 + 처벌 축 / 효율성 축 + 규칙 합법성 축). 5점 영역(각 논거에 반례·선제 방어 붙이기)은 Lv.3~4 진입 시 재훈련 예정 |

## 업데이트 프로토콜

### 언제 업데이트하는가

- `/save` 트리거 또는 세션 종료 시
- 3편 미만 세션은 노이즈로 간주, 업데이트하지 않는다

### 어떻게 업데이트하는가

**피드백이 발생한 경우 (Active 업데이트):**

1. 해당 기준이 이미 Active 테이블에 있으면 — 최근 발생 날짜 갱신, 누적 횟수 +1
2. 없는 새 패턴이면 — Active 테이블에 행 추가
3. `last_updated` 날짜 갱신

**개선된 경우 (Resolved 이동 조건):**

- Active 기준이 **3회 연속 4점 이상**이면 Resolved로 이동
- Resolved 이동 시: Active에서 행 삭제 → Resolved 테이블에 추가

### 판단 기준

| 상황 | 처리 |
|------|------|
| 해당 기준 4점 이상 | 연속 성공 카운트 +1 |
| 3점 이하 | 연속 성공 카운트 리셋 |
| 3회 연속 4점 이상 | Resolved로 이동 |
| 새로운 약점 발견 | Active에 추가 |

## 피드백 억제 규칙

| 티어 | 기준 | 활성화 조건 |
|------|------|-----------|
| 1순위 | R2 Focus, R5 Structure | 항상 피드백 |
