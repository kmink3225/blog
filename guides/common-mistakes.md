# 자주 발생하는 오류 패턴 (Bad → Corrected)

## 경어체 혼입

본문 중간에 무의식적으로 경어체가 섞이는 패턴:

```
BAD:
Bayesian Optimization은 탐색과 활용의 균형을 맞추는 기법입니다.
대리 모델을 사용하여 목적 함수를 근사합니다.

CORRECTED:
Bayesian Optimization은 탐색과 활용의 균형을 맞추는 기법이다.
대리 모델을 사용하여 목적 함수를 근사한다.
```

## 수동 번호 붙이기

`number-sections: true`가 자동 번호를 부여하므로 이중 번호 발생:

```
BAD:                              렌더링 결과:
## 1. 개요                        1 1. 개요
## 2. 방법론                      2 2. 방법론
### 2.1 전처리                    2.1 2.1 전처리

CORRECTED:                        렌더링 결과:
## 개요                           1 개요
## 방법론                         2 방법론
### 전처리                        2.1 전처리
```

## 수식 공백 누락

인라인 수식 양쪽에 공백이 없으면 Quarto가 수식을 인식하지 못할 수 있다:

```
BAD:
모수$\theta$의 사전 분포를 설정하고, 사후 분포$p(\theta|D)$를 계산한다.

CORRECTED:
모수 $\theta$ 의 사전 분포를 설정하고, 사후 분포 $p(\theta|D)$ 를 계산한다.
```

## 콘텐츠 축소

원본의 구체적 예시를 추상적 한 줄로 압축하는 패턴:

```
BAD (원본의 구체적 비교를 삭제):
구작과 신작 저품질은 동일한 구조적 문제를 가진다.

CORRECTED (원본의 구체적 비교를 보존):
구작은 형식+콘텐츠 둘 다 문제이고, 신작 저품질은 콘텐츠만 문제이다.
retrofit-post.md의 Step 1(형식 교정)은 구작에만 필요하고,
Step 3(콘텐츠 보강)은 둘 다에 필요하다.
```
