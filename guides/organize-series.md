# 시리즈 블로그 정리

블로그 시리즈를 체계적으로 정리할 때 따르는 패턴.

> **선행 조건**: 이 스킬을 실행하기 전에 `guides/info-search.md`의 병렬 탐색 프로토콜을 먼저 실행한다.

## 절차

### Step 1: 기존 파일 전체 읽기

현재 상태를 파악한다.

### Step 2: 문제 진단

중복, 잘못된 내용, 누락된 주제, 비체계적 구성을 식별한다.

### Step 3: 구조 설계

기초 → 핵심 → 심화 → 실전 순서로 배치한다.

### Step 4: 파일 처리

- 잘 된 파일은 유지
- 잘못된 파일은 재작성
- 누락된 주제는 추가

### Step 4-B: 중복 파일 통합 시 콘텐츠 보강 (필수)

신규 포스트가 구 포스트를 대체할 때, **구 포스트의 고유 콘텐츠를 신규 포스트에 반드시 통합**한다.

::: {.callout-important}
## 통합 전 콘텐츠 비교 원칙

구 포스트의 링크를 삭제하기 **전에** 다음을 수행한다:

1. 구 포스트와 신규 포스트를 **전체** 읽는다
2. 구 포스트에만 있는 고유 콘텐츠를 식별한다 (수식 유도, 구체적 예시, 코드, 응용 사례 등)
3. 고유 콘텐츠를 신규 포스트의 적절한 위치에 **한다 체로 재작성하여 보강**한다
4. 구 포스트의 내용이 다른 포스트(예: 관련 시리즈)에서 이미 다뤄지는 경우, 크로스 링크로 대체한다
5. 보강이 완료된 후에만 구 포스트의 index 링크를 삭제한다
:::

**"신규 포스트가 구 포스트를 완전히 대체한다"고 판단하기 전에, 구 포스트의 수식 유도, 코드 예시, 응용 사례 등 고유 콘텐츠가 정말로 신규 포스트에 포함되어 있는지 꼼꼼히 확인한다.** 단순히 주제가 같다고 해서 대체 완료로 간주하지 않는다.

### Step 5: 요약 테이블 제공

작업 결과를 사용자에게 보고한다. 통합한 경우 어떤 콘텐츠를 어디에 보강했는지 명시한다.
