# 기존 포스트 교정 (Retrofit)

> **사용 시점**: 사용자가 기존 포스트를 열어 현행 규칙에 맞게 고쳐달라고 요청할 때 실행한다.

## 배경

skill-based prompt 체계 도입 이전에 작성된 포스트는 현행 AGENT_GUIDE.md 규칙을 따르지 않는다.
이 스킬은 기존 포스트를 **기존 내용을 보존하면서** 현행 규칙에 맞게 교정한다.

---

## Step 0: 사전 준비

1. 대상 포스트를 **전체** 읽는다
2. 해당 카테고리의 `GUIDE.md`를 읽는다
3. 해당 카테고리의 `index.qmd`를 읽는다

---

## Step 1: 형식 교정 (기계적 수정)

형식 위반은 판단이 필요 없는 기계적 수정이다. 아래 체크리스트를 **위에서 아래로** 순서대로 처리한다.

### 1-A. YAML 헤더

| 점검 항목 | 위반 예시 | 수정 방법 |
|----------|----------|----------|
| 필수 필드 누락 | `author` 없음 | `author: Kwangmin Kim` 추가 |
| `description` 누락 또는 비어 있음 | `description: ""` | 본문을 읽고 1~3문장으로 작성 |
| `date` 형식 | `2024-08-01` (ISO) | `08/01/2024` (MM/DD/YYYY)로 변환 |
| `categories` 부족 | 주 카테고리만 있음 | 본문 내용에 맞게 관련 카테고리 추가 |
| 카테고리 GUIDE의 커스텀 필드 누락 | Governance의 `format.html.grid` 없음 | GUIDE.md의 YAML 템플릿에 맞춰 추가 |

### 1-B. 섹션 헤더 수동 번호 제거

가장 흔한 위반이다. 다음 패턴을 모두 탐지하여 제거한다:

```
# 탐지 패턴 (정규식)
^#{2,6}\s*<?b?>?\s*\d+[\.\)]\s*   ← ## 1. 제목, ### <b>2. 제목</b>, #### 3) 제목

# 변환 예시
## 1. 개요                    →  ## 개요
### <b>2. 텐서 소개</b>       →  ### 텐서 소개
#### 3) 세부 항목              →  #### 세부 항목
##### (1) 하위 항목            →  ##### 하위 항목
```

HTML 태그(`<b>`, `<br>`, `<font>` 등)가 헤더에 사용된 경우 함께 제거한다.
본문 내의 순서 목록(`1. 항목`, `2. 항목`)은 마크다운 ordered list이므로 건드리지 않는다.

### 1-C. 경어체 → 한다 체

본문에서 경어체 어미를 탐지하고 한다 체로 변환한다.

| 경어체 | 한다 체 |
|--------|--------|
| ~합니다 | ~한다 |
| ~입니다 | ~이다 |
| ~됩니다 | ~된다 |
| ~있습니다 | ~있다 |
| ~하세요 | ~한다 |
| ~겠습니다 | ~겠다 / 제거 |
| ~드립니다 | 제거 또는 재구성 |

::: {.callout-important}
## 맥락 주의

"합니다"가 인용문, 코드 주석, UI 텍스트 재현 등에서 사용된 경우는 원문 그대로 유지한다.
기계적으로 치환하지 말고, 각 문장의 맥락을 확인한 후 변환한다.
:::

### 1-D. 줄 개행 (trailing 공백 2칸)

본문에서 문단 내 줄바꿈이 의도된 곳에 trailing 공백 2칸이 있는지 확인한다.
다음은 **적용하지 않는** 영역이다:

- 빈 줄(`\n\n`)로 구분되는 문단 사이
- 코드 블록 내부
- YAML 헤더 내부
- callout 블록 내부
- bullet list 항목

### 1-E. LaTeX 공백

인라인 수식 `$...$`의 양쪽에 공백이 있는지 확인한다.

```
# 위반
모수$\theta$를 추정한다

# 수정
모수 $\theta$ 를 추정한다
```

### 1-F. 이모지 제거

본문에 이모지가 있으면 제거한다. 의미 전달에 필요한 경우 텍스트로 대체한다.

### 1-G. HTML 태그 정리

마크다운으로 대체 가능한 HTML 태그를 변환한다.

| HTML | 마크다운 대체 |
|------|-------------|
| `<b>텍스트</b>` | `**텍스트**` |
| `<i>텍스트</i>` | `*텍스트*` |
| `<br>` | 줄 끝 공백 2칸 또는 빈 줄 |
| `<code>텍스트</code>` | `` `텍스트` `` |
| `<h2>제목</h2>` | `## 제목` |

`<img>`, `<table>`, `<div>` 등 마크다운으로 대체하기 어려운 태그는 유지한다.

## Step 2: 구조 교정

### 2-A. 카테고리 GUIDE.md 콘텐츠 구조 대조

카테고리 GUIDE.md에 정의된 포스트 콘텐츠 구조(예: Governance의 경우 정의 → 개념 → 왜 필요한가 → 응용 → 예시 → 코드 → 관련 주제)와 대상 포스트의 실제 구조를 대조한다.

| 상황 | 조치 |
|------|------|
| GUIDE에 정의된 섹션이 포스트에 없음 | 추가 가능한 내용이면 추가, 해당 없는 내용이면 생략 |
| 포스트에 GUIDE에 없는 섹션이 있음 | 내용이 유효하면 유지 |
| 섹션 순서가 GUIDE와 다름 | GUIDE 순서에 맞게 재배치 |

### 2-B. 관련 주제 섹션

포스트 끝에 "관련 주제" 섹션이 없으면 추가한다.
GUIDE.md의 관련 주제 링크 패턴을 따르되, 실제 존재하지 않는 파일도 placeholder 링크로 포함한다.

### 2-C. index.qmd 정합성

대상 포스트가 해당 카테고리의 `index.qmd`에 링크되어 있는지 확인한다.

| 상황 | 조치 |
|------|------|
| 링크가 없음 | index.qmd의 기존 패턴에 맞게 링크 추가 |
| 링크가 있지만 형식이 다름 | 기존 패턴에 맞게 수정 |
| 크로스 카테고리 링크가 필요하지만 없음 | 관련 카테고리의 index.qmd에 크로스 링크 추가 |

---

## Step 3: 콘텐츠 품질 교정

형식과 구조가 정리된 후, 내용의 품질을 보강한다.
이 단계는 `guides/audit.md`의 감사 항목 A~G를 적용하되, 기존 포스트의 특성을 고려한 추가 규칙이 있다.

### 기존 포스트 특유의 문제 패턴

| 패턴 | 설명 | 보강 방법 |
|------|------|----------|
| **노트 수준 콘텐츠** | 메모장 수준의 짧은 서술, 맥락·배경 없음 | 각 섹션에 "왜 이것이 필요한가", "이것이 없으면 어떻게 되는가"를 추가 |
| **코드만 나열** | 코드 블록만 있고 설명이 없거나 한 줄 수준 | 코드 앞에 "이 코드가 하는 일"을, 코드 뒤에 "핵심 포인트"를 추가 |
| **용어 미정의 사용** | 전문 용어가 정의 없이 등장 | 첫 등장 시 괄호 풀이 또는 정의 문장 추가 |
| **표만 있고 해설 없음** | 비교 표나 분류 표 아래에 해석이 없음 | 표 아래에 "이 표에서 주목할 점은…" 또는 시나리오 기반 설명 추가 |
| **절차만 나열** | "1단계: X, 2단계: Y" 식으로 절차만 있고 왜 이 순서인지 없음 | 절차 전에 전체 흐름의 목적을, 각 단계에 "이 단계에서 ~를 확보한다" 추가 |

### 보강 원칙

- **기존 내용을 삭제하지 않는다** — 추가만 한다
- **원저자의 의도를 존중한다** — 구조를 과도하게 변경하지 않는다
- **보강 분량이 원문의 2배를 넘기면** 별도 포스트 분리를 검토한다 (`audit.md` Step 3 판단 기준표 참조)
- **정보 탐색이 필요하면** `guides/info-search.md`를 실행하여 교재·블로그 내 관련 콘텐츠를 확인한 후 보강한다

---

## Step 4: 감사 실행

모든 교정이 완료되면 `guides/audit.md`의 전체 절차(Step 1~5)를 실행한다.
이 감사에서 추가로 발견된 항목이 있으면 보강한다.

---

## Step 5: 변경 요약

교정 완료 후 사용자에게 변경 사항을 요약 보고한다.

```
## 교정 요약

**대상**: `파일경로`

### 형식 교정
- [ ] YAML: description 추가, date 형식 변환
- [ ] 헤더: 수동 번호 12건 제거
- [ ] 스타일: 경어체 → 한다 체 8건
- [ ] HTML: <b> 태그 5건 → 마크다운 변환

### 구조 교정
- [ ] "관련 주제" 섹션 신규 추가
- [ ] index.qmd 링크 추가

### 콘텐츠 보강
- [ ] 섹션 "X": 직관적 설명 추가 (감사 항목 A)
- [ ] 섹션 "Y": 시나리오 기반 예시 추가 (감사 항목 B)
- [ ] 별도 포스트 분리: 없음 / `파일경로` 생성
```

## 효율 팁

### 대량 교정 시

사용자가 여러 포스트를 연속으로 교정 요청할 때:

- 같은 카테고리 내 포스트는 GUIDE.md를 한 번만 읽는다
- Step 1(형식 교정)은 패턴이 동일하므로 빠르게 처리한다
- Step 3(콘텐츠 보강)에 시간을 집중한다

### 교정 깊이 조절

사용자가 교정 범위를 지정할 수 있다:

| 사용자 요청 | 실행 범위 |
|------------|----------|
| "형식만 맞춰줘" | Step 1만 실행 |
| "구조까지 정리해줘" | Step 1 + Step 2 |
| "내용도 보강해줘" / 범위 미지정 | Step 1 + Step 2 + Step 3 + Step 4 (전체) |

### 수정하지 않는 것

- **파일명**: 기존 포스트의 파일명은 이미 URL로 공유되었을 수 있으므로 변경하지 않는다
- **date**: 원래 작성일을 유지한다 (형식만 변환)
- **기존 내용 삭제**: 명백한 오류가 아닌 한 기존 내용을 삭제하지 않는다
- **`_quarto.yml`, `_metadata.yml`**: 프로젝트 설정 파일은 건드리지 않는다
