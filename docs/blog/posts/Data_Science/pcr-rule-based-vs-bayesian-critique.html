개요

진단 시스템의 신호 처리 알고리즘은 크게 두 패러다임으로 구분된다.

규칙 기반(Rule-Based) 알고리즘은 도메인 전문가의 지식을 명시적 조건과 분기로 표현한다. 신호가 특정 조건을 만족하면 특정 처리를 수행하고, 그 결과를 다음 단계로 넘기는 순차적 구조다.

데이터 기반(Data-Driven) 알고리즘은 데이터로부터 직접 파라미터를 추정한다. 규칙을 사전에 정의하는 대신, 통계 모델이 데이터의 패턴을 학습하여 추정과 불확실성을 함께 제공한다.

두 접근법은 대립하는 것이 아니라 서로 다른 문제를 잘 푼다. 진단 시스템 설계에서 어떤 접근법이 어떤 조건에서 적합한지를 이해하는 것이 핵심이다.

왜 <1%의 gray zone이 중요한가

PCR은 강력한 증폭 메커니즘 덕분에 대다수 신호에서 정확하고 신뢰할 수 있는 추정을 제공한다. 명확한 양성과 음성 신호에서 규칙 기반 파이프라인은 충분히 안정적으로 동작한다.

문제는 모든 기술이 완벽하지 않듯, 노이즈가 강하게 섞인 약 1%의 gray zone 신호다. 비율만 보면 작아 보이지만, 절대 수치는 다르다.

총 샘플 수 gray zone 비율 해당 샘플 수
1,000명 <1% <10명
10,000명 <1% <100명
100,000명 <1% <1,000명

10,000명 규모의 검사에서 100명이 gray zone에 해당한다. 이 100명에 대한 판정 오류(위양성 또는 위음성)는 임상적으로 무시할 수 없는 수준이다. 규칙 기반 파이프라인이 99%에서 충분히 잘 동작한다는 사실이 나머지 1%를 방치하는 근거가 되지 않는다.

데이터 기반 접근법이 전체 파이프라인을 대체하는 것을 목표로 하지 않는 이유도 여기에 있다. 잘 동작하는 99%는 규칙 기반이 처리하고, 해석이 필요한 1%에 집중적으로 추가 분석을 적용하는 것이 실용적 목표다.

이 risk는 의사에게 판정 근거를 제공하여 재검을 권장하는 방식으로 완화할 수 있다. “Ct가 34.9로 임계값 근처입니다. 증폭 효율이 낮아 신뢰도가 낮습니다. 재검을 권장합니다.”와 같은 설명이 가능하다면, gray zone 신호에 대한 불확실성 관리가 가능하다.

규칙 기반 알고리즘

구조

규칙 기반 파이프라인은 신호를 순서대로 변형하고 각 단계에서 조건 분기를 적용한다.

원본 신호 (N개 데이터 포인트)
    ↓ Step 1: 조건 A 판단 → 처리 방식 1 또는 2 선택
    ↓ Step 2: 조건 B 판단 → 이상 신호 플래그 처리
    ↓ Step 3: 정규화
    ↓ Step 4: 파라미터 모델 fitting
    ↓ 파라미터 → 임계값 비교 → 판정

강점

투명성과 추적 가능성: 각 단계의 조건이 명시적이므로 판정 경로를 코드 수준에서 추적할 수 있다.

속도: 조건 분기와 선형 연산으로 구성되어 실시간 처리에 적합하다. C++ 구현 기준으로 수백 개 신호를 수십 ms 내에 처리한다.

규제 대응 용이성: FDA 등 규제 기관은 알고리즘의 결정 경로를 검증 가능한 형태로 요구한다. 규칙 기반은 각 조건과 임계값을 문서화하기 쉽다.

도메인 지식 직접 반영: 전문가가 축적한 경험적 지식을 알고리즘에 직접 내재화할 수 있다.

한계

오류 전파: 각 단계의 처리 오차가 다음 단계로 누적된다. 최종 파라미터가 어느 단계의 오류를 얼마나 반영하는지 역추적이 어렵다.

임계값의 경직성: 조건 분기의 임계값은 개발 데이터셋에서 경험적으로 결정된다. 장비 lot 변경, 시약 조성 변화, 새로운 샘플 유형 등 환경 변화에 취약하다.

불확실성 정량화의 어려움: 규칙 기반 판정은 이진(양성/음성) 출력이 기본이다. gray zone 신호에서 “얼마나 불확실한가”를 수치로 제공하기 어렵다.

설명력의 한계: 판정 결과와 함께 제공할 수 있는 해석이 알고리즘의 분기 경로에 한정된다. 사용자(예: 의료 전문가)에게 신호의 물리적·생물학적 특성에 기반한 납득 가능한 근거를 제공하기 어려울 수 있다. “조건 X를 만족하여 음성 처리”는 “왜 그 조건인가”라는 질문에 답하지 못한다.

규칙 기반이 제공하는 해석의 예시와, 전문가가 실제로 원하는 정보를 비교하면 다음과 같다.

항목 규칙 기반이 제공하는 것 전문가가 원하는 것
음성 이유 “기울기 0.8 이하라서 음성 처리” “이 샘플에 타겟이 없다는 근거”
양성 근거 “Ct = 28.3 (임계값 35 이하)” “신호 품질, 효율, 신뢰도 수준”
불확실 케이스 “판정 불가 — 재검” “왜 불확실한가, 무엇이 문제인가”

이 간극은 규칙 기반 알고리즘의 결함이 아니라 설계 철학의 차이에서 비롯된다. 규칙 기반은 일관된 처리 절차를 보장하는 데 최적화되어 있고, 개별 판정에 대한 확률적 해석을 제공하도록 설계되지 않았다.

데이터 기반 알고리즘

구조

데이터 기반 접근법은 원본 신호에서 모든 성분을 동시에 추정한다. Bayesian joint estimation을 예로 들면:

원본 신호 (N개 데이터 포인트)
    ↓ 동시 추정 (joint)
    ├── baseline 성분
    ├── 이상 신호(jump) 성분: sparse prior로 자동 선택
    └── sigmoid 성분: A, D, C, B
    ↓
각 파라미터의 posterior 분포 (추정값 + 불확실성)

전처리가 분리된 단계가 아니라 모델의 일부가 된다. 파라미터는 변형된 신호가 아닌 원본 신호에서 직접 추정된다.

강점

불확실성의 명시적 정량화: posterior 분포에서 신뢰 구간, 확률적 판정이 자동으로 산출된다.

# 어떤 신뢰 수준이든 posterior에서 직접 계산
ci_95 = np.percentile(ct_samples, [2.5, 97.5])
p_positive = (ct_samples < threshold).mean()   # P(양성 | 데이터)

파라미터 기반 해석: 각 파라미터가 신호의 물리적 특성에 대응하므로 판정 근거를 신호 특성의 언어로 설명할 수 있다.

"증폭 효율(B) = 74%로 정상 범위 이하입니다.
 S자 곡선 전환점(Ct)의 95% 신용구간이 7.2 cycle로 넓어
 위치를 특정할 수 없습니다.
 양성 확률: 64% — 재검을 권장합니다."

오류 격리: joint model에서 baseline 추정 오차, jump 성분, sigmoid 파라미터가 각각 독립적인 posterior를 가지므로 어느 성분이 판정 불확실성에 기여하는지 추적할 수 있다.

환경 변화 적응: Hierarchical Bayesian을 사용하면 배치 수준 파라미터(예: 증폭 효율의 lot별 평균)가 데이터로부터 자동으로 갱신된다.

한계

계산 비용: MCMC 기반 posterior 추정은 신호 1개당 수백 ms가 소요된다. 수백 개 신호의 실시간 처리에는 적합하지 않다.

실시간 처리가 요구되는 환경에서 속도 문제를 해결하는 방법들이 있다. Variational Inference(VI)는 MCMC보다 수십 배 빠르지만 posterior 근사 정확도가 낮아진다. Laplace Approximation은 더 빠르지만 posterior 형태가 단순한 경우에만 유효하다. JAX/GPU를 활용한 병렬 MCMC는 N개 신호를 일괄 처리할 수 있어 총 처리 시간을 줄인다.

방법 신호 1개 처리 시간 정확도
MCMC (NUTS) 수백 ms 높음
Variational Inference 수십 ms 중간
Laplace Approximation 수 ms 낮음 (단순 posterior 가정)
Bootstrap (Frequentist) 수십 ms 중간

Prior 설정 의존성: prior가 잘못 설정되면 결과도 틀린다. 충분한 도메인 지식과 검증 데이터가 필요하다.

규제 검증 경로: Bayesian posterior 기반 판정 시스템의 규제 승인 경로는 규칙 기반보다 복잡하다. 검증 프레임워크를 새로 구축해야 한다.

Prior 민감도 분석 필요: Prior가 결과에 미치는 영향을 규제 관점에서 문서화해야 한다. Prior를 바꾸면 posterior가 얼마나 달라지는지, 어떤 prior 범위에서 판정이 안정적인지를 입증해야 한다.

두 접근법이 실패하는 조건

각 접근법이 취약한 조건을 이해하면 Hybrid 설계의 근거가 명확해진다.

규칙 기반이 실패하는 조건

Edge case와 gray zone: 임계값 근방의 신호는 규칙의 분기 방향에 따라 판정이 크게 달라진다. Ct = 34.9와 Ct = 35.1은 물리적으로 유사하지만 규칙 기반에서 반대 판정이 나온다.

환경 변화: 시약 lot이 바뀌어 증폭 효율이 전반적으로 낮아지면 기존 임계값이 무효가 된다. 규칙 재조정 없이는 위음성이 증가한다.

복합 이상 신호: 여러 종류의 노이즈가 동시에 발생하면 각 이상을 처리하는 규칙들이 서로 충돌하거나 순서 의존성 문제가 발생한다.

데이터 기반이 실패하는 조건

데이터 부족: 단일 신호에 45개 데이터 포인트로 57개 파라미터를 추정하려 하면 과적합이 발생한다. L1 prior 등 정규화 장치가 없으면 모델이 수렴하지 않는다.

Prior 오설정: 도메인 지식이 부정확하게 반영된 prior는 posterior를 왜곡한다. 특히 데이터가 적은 gray zone 신호에서 prior 영향이 크다.

Out-of-distribution 신호: 학습 데이터에 없는 패턴의 신호에 대해 posterior가 지나치게 좁게 추정될 수 있다.

핵심 차이: 전처리 후 파라미터 해석

두 접근법의 가장 중요한 차이 중 하나는 파라미터 해석의 유효성이다.

규칙 기반 파이프라인에서 4PL 파라미터는 여러 단계의 전처리가 적용된 잔여 신호에서 fitting된다.

\[ \text{원본 신호} \xrightarrow{\text{Step 1, 2, 3}} \text{변형된 신호} \xrightarrow{\text{4PL fitting}} B = 0.37 \]

이 B = 0.37에는 Step 1~3의 처리 오차가 얼마나 포함되어 있는지 알 수 없다. 이 값을 “증폭 효율”로 해석하는 것은 근사적으로는 맞을 수 있지만, 엄밀하게는 전처리된 잔여 신호의 fitting 결과다.

데이터 기반 joint model에서 동일한 B는 원본 신호에서 직접 추정된다.

\[ \text{원본 신호} \xrightarrow{\text{joint estimation}} B = 0.24 \pm 0.05 \text{ (posterior)} \]

이 경우 B = 0.24는 원본 신호의 증폭 특성에 더 가까우며, 불확실성 \(\pm 0.05\) 도 함께 제공된다.

공식 문서화와 불확실성 추적의 차이

전처리 절차가 공식 문서에 명시되어 있는 것과, 그 전처리가 최종 파라미터 추정의 불확실성에 얼마나 기여하는지를 추적하는 것은 다른 차원의 문제다.

  • 문서화 = 절차의 투명성 (어떤 처리를 했는가)
  • 불확실성 추적 = 결과의 신뢰성 (처리 오차가 결과에 얼마나 영향을 미쳤는가)

업계 표준 전처리 방식을 채택한다고 해서 불확실성 추적이 자동으로 해결되지는 않는다.

Hybrid 아키텍처 설계

두 접근법의 한계를 고려하면, 실용적 해법은 각각의 강점을 활용하는 Hybrid 구조다.

전체 신호 (N개)
    ↓
규칙 기반 (실시간, < 10ms)
    ├── 명확 양성 (~40~45%) → 결과 즉시 출력
    ├── 명확 음성 (~40~45%) → 결과 즉시 출력
    └── gray zone (~10~20%) → 데이터 기반 추가 분석
                                ↓
                      posterior 기반 불확실성 정량화
                      신뢰도 점수 + 파라미터 해석 추가

gray zone 신호만 데이터 기반으로 처리하면 전체 지연을 현실적인 수준으로 유지할 수 있다. 96웰 기준 gray zone 15%면 약 115개 신호만 추가 처리 대상이 된다.

속도 분석

전체 신호를 순차적으로 MCMC 처리하는 것과 Hybrid 구조를 비교하면:

방식 처리 대상 총 처리 시간
전체 MCMC 768개 신호 × 300ms ~4분
Hybrid (규칙 + MCMC) gray zone ~115개 × 300ms ~35초
Hybrid (규칙 + VI) gray zone ~115개 × 30ms ~3.5초
Hybrid (규칙 + JAX/GPU) gray zone 115개 일괄 510초

현실적인 구현에서 gray zone 신호를 비동기로 처리하면 추가 지연이 임상 운영에 미치는 영향을 최소화할 수 있다. 규칙 기반 판정을 먼저 출력하고, 데이터 기반 신뢰도 분석이 완료되면 업데이트하는 방식이다.

출력 형태

데이터 기반 레이어의 역할은 규칙 기반 판정을 번복하는 것이 아니라 신뢰도 정보를 추가하는 것이다.

[규칙 기반 판정]
  결과: 양성

[데이터 기반 신뢰도 분석]
  증폭 효율 (B): 74%  [정상: 90~110%]

Bayesian Credible Interval의 직접 해석

데이터 기반 접근법의 실용적 이점 중 하나는 신뢰 수준 표현 방식이다. Frequentist 95% 신뢰구간(CI)은 엄밀히 “이 방법으로 반복 계산하면 95%는 참값을 포함한다”는 의미다. 파라미터 자체가 이 구간 안에 있을 확률을 직접 말할 수 없다.

Bayesian 신용구간(Credible Interval)은 posterior 샘플에서 percentile로 계산하여 직접적인 확률 해석이 가능하다.

ct_samples = trace.posterior["C"].values.flatten()

ci_80 = np.percentile(ct_samples, [10,  90])    # "Ct가 이 구간에 있을 확률 80%"
ci_95 = np.percentile(ct_samples, [2.5, 97.5])  # "Ct가 이 구간에 있을 확률 95%"
ci_99 = np.percentile(ct_samples, [0.5, 99.5])  # "Ct가 이 구간에 있을 확률 99%"

p_positive = (ct_samples < threshold).mean()    # P(Ct < threshold | 데이터)

어떤 신뢰 수준이든 percentile 한 줄로 계산된다. 파라미터 간 결합 확률도 직접 계산할 수 있다.

b_samples  = trace.posterior["B"].values.flatten()

# "Ct < 35이면서 증폭 효율 > 90%일 확률" — Frequentist로는 매우 어려운 계산
p_joint = ((ct_samples < 35) & (b_samples > 0.279)).mean()

이 방식으로 전문가에게 “이 판정의 확률적 근거”를 수치로 제공할 수 있다.

이행 경로

규칙 기반 시스템을 즉시 교체하는 것은 현실적이지 않다. 규제 승인, 검증 데이터 구축, 운영 안정성 확보 모두 시간이 필요하다. 단계적 이행이 현실적이다.

Phase 1 — 신뢰도 레이어 추가

기존 파이프라인을 유지하면서 gray zone 신호에 한해 데이터 기반 분석을 병렬 실행한다. 두 방법의 결과 불일치 케이스를 수집하여 실제 위양성·위음성 패턴을 확인한다.

Phase 2 — Semi-Joint 전환

4PL fitting을 Bayesian으로 전환하고, baseline과 이상 신호 처리는 규칙 기반을 유지한다. 불확실성 정량화가 파이프라인에 포함된다.

Phase 3 — Full Joint

충분한 검증 데이터와 규제 근거가 확보되면 Full Bayesian joint로 전환한다. 파라미터 해석이 원본 신호 특성을 직접 반영한다.

이행 원칙

각 Phase의 전환 조건을 데이터로 정의해야 한다. 예를 들어 “Phase 1 결과에서 gray zone 신호의 재검 일치율이 X% 이상이면 Phase 2로 전환”과 같은 정량적 기준이 없으면 이행이 무기한 지연된다. 전환 기준을 미리 합의하는 것이 프로젝트 관리의 핵심이다.

이 이행 경로는 PCR 진단 시스템에 한정되지 않는다. 규칙 기반으로 운영되는 진단 시스템 전반 — 심전도 이상 탐지, 의료 영상 품질 관리, 환경 모니터링 센서 신호 분석 — 에서 동일한 구조로 적용할 수 있다. 핵심은 잘 동작하는 대다수를 규칙 기반으로 유지하면서, 해석이 필요한 소수에 집중적으로 데이터 기반 분석을 적용하는 것이다.
